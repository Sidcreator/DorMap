Изменение 2: WHERE в справочнике статусов документов

Проблема

Справочник CG_Online_СтатусыДокументовСервисов не заполнялся при нажатии кнопки "Создать статусы". Без элемента с ТипСтатуса = Ошибка запрос СтатусОшибка (стр. 3227–3234 в ДозаполнитьЗаказыКлиентовСервисов) возвращал 0 строк → Result[8] = ЗаказыКСозданию пуст → заказы не создавались.

Причина

В пакетном запросе метода СтатусыДокументовСервисовПоНастройке Part 2 заполняет временную таблицу СтатусыДокументовСервисов из существующих данных справочника. При пустом справочнике — 0 строк. Part 3 делает LEFT JOIN с пустой таблицей → все поля СтатусыДокументовСервисов = NULL. WHERE на стр. 601 проверял НЕ СтатусыДокументовСервисов.Наименование ЕСТЬ NULL → NULL = ЛОЖЬ → все строки отфильтрованы.

Исправление

Файл: Catalogs\CG_Online_СтатусыДокументовСервисов\Ext\ManagerModule.bsl

Метод: СтатусыДокументовСервисовПоНастройке (стр. 502)

// Было (строка 601): И НЕ СтатусыДокументовСервисов.Наименование ЕСТЬ NULL // Стало (строка 601): И НЕ ТаблицаСтатусы.Заголовок ЕСТЬ NULL
