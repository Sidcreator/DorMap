Метаданные для записи заказов Ozon в CG_Online

При загрузке заказа с Ozon (FBS/FBO) данные проходят через 19+ объектов метаданных: 2 документа, 15 регистров, 2 справочника, 3 перечисления, а также 4 макета для начальной загрузки статусов.

9. Загрузка статусов — пути и проблема

Полная цепочка загрузки статусов

Кнопка "Создать статусы" (CG_Online_СтатусыДокументовСервисов.СоздатьСтатусыДокументовСервисов)
  → CG_Online_ЗначениеНастроек.СоздатьСтатусыДокументовСервисов()
    → Справочники.Менеджер.СоздатьСтатусыДокументовСервисов()
      → CommonModule.ЗагрузитьВсеОбъекты()
        → Есть настройки авторизации?
          │
          ├── ДА → ЗагрузитьОбъекты() для каждой настройки
          │         │
          │         ├── Коннектор = Неопределено? → ВОЗВРАТ (тихо)
          │         │
          │         └── Коннектор определён:
          │                │
          │                ├── Источник = Макет?
          │                │     │
          │                │     ├── Connector = ОбщийМодуль?
          │                │     │     └── ДА → ЗагрузитьСтатусыДокументовСервисовПоНастройке()
          │                │     │             → проверяет подсистемы (ИСТП, ТП, ИССОХ, ИСП)
          │                │     │             → загружает из макетов справочника
          │                │     │
          │                │     └── НЕТ → Коннектор.Метаданные().Макеты.Найти("СтатусыДокументовСервисов")
          │                │                  │
          │                │                  └── Не найден? → ВОЗВРАТ (тихо! БАГ)
          │                │
          │                └── Источник ≠ Макет → ВыполнитьИнтеграционнуюФункцию()
          │                                      → Ozon: Коннектор.ПолучитьМакет("СтатусыДокументовСервисов")
          │
          └── НЕТ → ЗагрузитьСтатусыДокументовСервисовПоНастройке()
                     → загружает из макетов справочника

Известный баг: кнопка "Создать статусы" не работает для Ozon

Причина: В ЗагрузитьОбъекты() (строка 378) проверка макета делается через Коннектор.Метаданные().Макеты.Найти("СтатусыДокументовСервисов").

Коннектор Ozon — это ссылка на ДополнительныеОтчетыИОбработки. Метаданные() возвращает метаданные справочника, а не внешней обработки. У справочника нет макета "СтатусыДокументовСервисов" → Макет = Неопределено → тихий возврат.

При реальной синхронизации заказов вызывается ВыполнитьИнтеграционнуюФункцию("ЗагрузитьСтатусыДокументовСервисов"), который корректно попадает в Ozon-модуль и загружает макет через Коннектор.ПолучитьМакет("СтатусыДокументовСервисов").

Workaround: Создать элемент "Ошибка" вручную через код.
