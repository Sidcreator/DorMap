2.3. Описание функций УОТ609

ЗагрузитьЗаказы — строка 5327

Обёрточная функция. Вызывает два загрузчика последовательно:

  1. ЗагрузитьЗаказыССобственногоСклада — FBS-заказы
  2. ЗагрузитьЗаказыСоСкладаСервиса — FBO-заказы

ЗагрузитьЗаказыССобственногоСклада — строки 4774–5108

Основная функция загрузки FBS-заказов. Выполняет:

ШагОписаниеAPI
1Инициализация таблиц: тзТовары, тзЗаказы, тзШтрихкода—
2Цикл пагинации с step=50v3/posting/fbs/list
3Обработка каждого заказа из result.postings—
4Заполнение тзТовары: SKU, количество, цена—
5Заполнение тзЗаказы: номер, статус, сумма, данные логистики—
6Заполнение тзШтрихкода: штрихкоды упаковок—
7Обработка financial_data: комиссии, скидки, логистика—
8Вызов ЭкзмеплярыТоваровПоОтправлениямv6/fbs/posting/product/exemplar/create-or-get
9Вызов CG_Online_ИнтеграцииССервисамиСервер.ЗагрузитьЗаказыКлиентовСервисовПоНастройке—

Ключевые параметры:

  • Передаёт таблицы тзТовары, тзЗаказы, тзШтрихкода как отдельные параметры
  • Использует CG_Online_ИнтеграцииССервисамиСервер.СтатусЗаказаКлиентаСервисаПоИдентификатору для определения статуса

ОбновитьСтатусЗаказа — строки 5465–5526

Обновление статуса заказа через API. Выполняет:

ШагОписание
1Получение данных заказа
2Формирование структуры packages с грузовыми местами
3Формирование структуры products с товарами
4Отправка v4/posting/fbs/ship
5Обработка ответа

Особенность: Простая версия без поддержки exemplar_info.

ЭкзмеплярыТоваровПоОтправлениям — строка 6179

Получение данных маркировки товаров. Использует API v6/fbs/posting/product/exemplar/create-or-get.

СинхронизироватьСтатусыЗаказовКлиентовСервисов — строки 11031–11034

Заглушка. Вызывает ЭкранированиеНеРеализованногоФункционала — функционал синхронизации статусов не реализован в УОТ609.
