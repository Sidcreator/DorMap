Отчёт: Параметры, влияющие на ответы запросов при загрузке заказов Ozon

Полный каталог настроек и флагов, которые задаются в клиенте (справочники, константы, функциональные опции) и влияют на то, какие данные попадают в итоговый результат загрузки заказов Ozon.

1. Сквозная диаграмма фильтрации (Gates)

flowchart TD A["Фаза A: ЗагрузитьЗаказыССобственногоСклада()"] --> G0{"Gate 0: Параметры API"} G0 --> G1{"Gate 1: Процессинг"} G1 -->|TRUE| REG["Регистры (обход)"] G1 -->|FALSE| G2{"Gate 2: Складской фильтр"} G2 --> G3{"Gate 3: ВД фильтр"} G3 --> G4{"Gate 4: Identity Map"} G4 --> G5{"Gate 5: НоменклатураСервисов"} G5 --> G6{"Gate 6: НовыеЗаказы"} G6 --> G7{"Gate 7: НастройкаГенерацииДокументов"} G7 --> DOCS["Создание документов ЗКС"]

7. Gate 5: Сопоставление с НоменклатураСервисов

Где: CG_Online_ЗаказКлиентаСервиса\Ext\Module.bsl:3381–3476 (первичный) и 3575–3589 (вторичный JOIN в СопоставленныеЗаказыСервисов)
Действие: Определяет, найден ли товар (номенклатура) для каждой строки заказа. Если товар не найден → строка уходит в ЗаказыСОшибкойТоваров.
ПараметрИсточникТипВлияние
Поля для поиска номенклатуры (LEFT JOIN с приоритетами):

ИдентификаторТовара Данные заказа + НоменклатураСервисов Строка Поиск: ВД.ИдентификаторТовара = НоменклатураСервисов.Идентификатор
Артикул Данные заказа + НоменклатураСервисов Строка Поиск: ВД.Артикул = НоменклатураСервисов.Артикул

ИдентификаторНоменклатурыСервиса Данные заказа + НоменклатураСервисов Строка Поиск по сервисному ID номенклатуры
ИдентификаторХарактеристикиСервиса Данные заказа + НоменклатураСервисов Строка Поиск по сервисному ID характеристики
ИдентификаторУпаковки Данные заказа + НоменклатураСервисов Строка Поиск по упаковке
ИдентификаторСерии Данные заказа + НоменклатураСервисов Строка Поиск по серии
Системные фильтры НоменклатураСервисов:
НоменклатураСервисов.Недействителен НоменклатураСервисов Булево Должен быть ЛОЖЬ — недействительные записи не участвуют в поиске
НоменклатураСервисов.ЭтоГруппа НоменклатураСервисов Булево Должен быть ЛОЖЬ — группы не участвуют в поиске товаров
НоменклатураСервисов.Владелец НоменклатураСервисов Справочник Равен ВД.Владелец (который берётся из НастройкаАвторизации.Партнер)
Флаг для авто-загрузки номенклатуры:
СоздаватьНоменклатуруПриЗагрузкеДокументов НастройкиАвторизации / НастройкиГенерацииДокументов Булево Если Ложь (по умолчанию) — товары из API НЕ загружаются в справочник. Тогда НоменклатураСервисов может быть пуста для новых Артикулов → все заказы уходят в ТоварыСОшибкой. Нужно включить в НастройкеАвторизации или НастройкеГенерации. 
-- Gate 5: Результат
ГДЕ (СопоставленныеЗаказыСервисов.Товар ЕСТЬ NULL
        ИЛИ СопоставленныеЗаказыСервисов.Товар = ЗНАЧЕНИЕ(Справочник.CG_Online_НоменклатураСервисов.ПустаяСсылка))
