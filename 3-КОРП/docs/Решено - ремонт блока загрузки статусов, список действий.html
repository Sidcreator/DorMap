Список действий

Действие 1: Анализ причины пустого справочника

Проблема: Кнопка "Создать статусы" в справочнике CG_Online_СтатусыДокументовСервисов не создаёт элементы. Справочник остаётся пустым. Без элемента ТипСтатуса = Ошибка запрос СтатусОшибка в ДозаполнитьЗаказыКлиентовСервисов (строка 3227–3234) возвращает 0 строк → Result[8] = ЗаказыКСозданию пуст → заказы не загружаются.

Сравнение: Проведён анализ двух процессов — заполнения СтатусыМодерацииТоваровСервисов (работает) и СтатусыДокументовСервисов (не работает).

Действие 2: Исправление WHERE (строка 601)

Файл: Catalogs\CG_Online_СтатусыДокументовСервисов\Ext\ManagerModule.bsl

Метод: СтатусыДокументовСервисовПоНастройке (строка 502)

Баг: WHERE в Part 3 запроса проверял поле временной таблицы, а не входных данных.

// Было (строка 601) — проверяет существующий справочник:
И НЕ СтатусыДокументовСервисов.Наименование ЕСТЬ NULL

// Стало (строка 601) — проверяет входную таблицу:
И НЕ ТаблицаСтатусы.Заголовок ЕСТЬ NULL

Механизм бага:

  1. Part 2 запроса: ПОМЕСТИТЬ СтатусыДокументовСервисов — читает существующие данные справочника
  2. При пустом справочнике Part 2 возвращает 0 строк
  3. Part 3: ТаблицаСтатусы LEFT JOIN СтатусыДокументовСервисов — все поля правой таблицы = NULL
  4. WHERE: НЕ NULL = ЛОЖЬ → все строки отфильтрованы
  5. Цикл записи не выполняется ни разу → справочник остаётся пустым

Действие 3: Запуск заполнения

После исправления нажать кнопку "Создать статусы" в панели администрирования УОТ:

  • Путь: Администрирование → CG_Online → Панель справочников интеграции с сервисами → Статусы документов сервисов → Создать статусы
  • Результат: Справочник заполняется элементами из макетов для всех сервисов

Схема запроса (исправленная)

Part 1: ПОМЕСТИТЬ ТаблицаСтатусы FROM &ТаблицаСтатусы          [входные данные из макета]
Part 2: ПОМЕСТИТЬ СтатусыДокументовСервисов FROM справочник    [существующие элементы]
  WHERE ПометкаУдаления = ЛОЖЬ

Part 3 (первый SELECT):
  FROM ТаблицаСтатусы
    LEFT JOIN СтатусыДокументовСервисов (cross-reference)
    LEFT JOIN СтатусыДокументовСервисовСУчетомСервиса (по сервису)
  WHERE
    НЕ ТаблицаСтатусы.Сервис ЕСТЬ NULL
    И НЕ ТаблицаСтатусы.ХозяйственнаяОперация ЕСТЬ NULL
    И НЕ ТаблицаСтатусы.ТипСтатуса ЕСТЬ NULL
    И НЕ ТаблицаСтатусы.Заголовок ЕСТЬ NULL        ← ИСПРАВЛЕНО
  UNION ALL
  SELECT FROM СтатусыДокументовСервисов              [элементы на удаление]
    LEFT JOIN ТаблицаСтатусы
  WHERE ТаблицаСтатусы.Идентификатор ЕСТЬ NULL
