> **Назначение**: Пошаговая инструкция для создания системы управления задачами с дорожной картой в любом проекте, используя Roo Code (AI-ассистент в VS Code).

> **Версия**: 3.0

## Обзор системы

Система состоит из 3 компонентов:

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **Дорожная карта** | `ДорожнаяКарта.md` | Файл с задачами, этапами, статусами, временем (минуты) |
| **Правила AI** | `.roo/rules` | Инструкции для AI-ассистента (ключевые фразы) |
| **Журнал изменений** | `ЖурналИзменений.md` | История всех изменений с детализацией |

```
проект/
```

## Шаг 2. Создать файл дорожной карты

Создать файл `docs/ДорожнаяКарта.md` (или в корне проекта) со следующей структурой:

### Обязательные секции:

### Конвенция: Лимит активных задач

- **Максимум 5 активных задач** (не отложенных) в дорожной карте
- При создании 6-й задачи без явного указания приоритета → **автоматически отложить** (перенести в секцию 7)
- Причина автооткладывания: «Превышен лимит активных задач (максимум 5)»
- Если одна из 5 задач закрыта (✅) → можно вытащить задачу из отложенных

### Конвенция: Отложенные задачи

- В **уровне 1** (дор карта 1) отложенные задачи отображаются **одной строкой**: `| **7** | ⏸️ Отложенные задачи (N шт.) |`
- В **уровне 2** (дор карта 2 / «статус задачи 7») — список всех отложенных задач с нумерацией `7.1`, `7.2`, `7.3`...
- В **уровне 3** — этапы каждой отложенной задачи: `7.1.1`, `7.1.2`...
- При команде «статус задачи 7» AI выдаёт все отложенные задачи с причинами и приоритетами
- При добавлении новой отложенной задачи: увеличить счётчик в уровне 1, добавить строку 7.N в уровень 2

### Конвенция: Учёт времени в минутах

- **Единица измерения:** минуты (целое число)
- **Колонка «Время (мин)»** — фактическое время, затраченное на задачу/этап
- **Правила заполнения:**
  - При старте задачи/этапа: `—` (прочерк)
  - При завершении: вписать количество минут (например: `15`, `120`, `5`)
  - При промежуточном статусе: `~N мин` (приблизительная оценка, если точное время неизвестно)
- **Итог по времени** — секция в конце дорожной карты:
  - Суммарное время по каждой задаче
  - Суммарное время за период (день/неделя)
  - Формат: `N мин` или `N ч M мин` при N ≥ 60
- **Команда «итого время»** — AI автоматически суммирует все значения из колонки «Время (мин)» и выводит сводку
- **Автоматический подсчёт:** при закрытии задачи AI предлагает вписать затраченное время или оценить его

```markdown
# Дорожная карта проекта [ИМЯ]

## Структура нумерации
| Уровень | Формат | Пример | Описание |
|---------|--------|--------|----------|
| 1 (задача) | N | 1 | Крупная задача |
| 2 (этап) | N.N | 1.1 | Стадия выполнения |
| 3 (шаг) | N.N.N | 1.1.1 | Конкретное действие |

## Сводная таблица задач (Уровень 1)
| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
|---|--------|---|-----------|---|--------|--------|-------|-------------|
| 1 | [Название] | P1 | ⭐ | 0% | 📋 | — | — | — |

## Детализация по задачам (Уровни 2 и 3)
### 1. [Название задачи] 📋
| № | Этап | Описание | Статус | Дата | Время (мин) | Журнал |
|---|------|----------|--------|------|-------------|--------|
| 1.1 | [Этап] | [Описание] | ⬜ | | — | |

## 6. ✅ Выполненные задачи
> В уровне 1 — одна строка: `| **6** | ✅ Выполненные задачи (N шт.) | — | — | — | ✅ | — | — | — |`

| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
|---|--------|---|-----------|---|--------|--------|-------|-------------|

## 7. ⏸️ Отложенные задачи
> В уровне 1 — одна строка: `| **7** | ⏸️ Отложенные задачи (N шт.) | — | — | — | ⏸️ | — | — | — |`

| № | Задача | Причина | P | Дата |
|---|--------|---------|---|------|

### 7.N [Название отложенной задачи] ⏸️
| № | Этап | Описание | Статус | Дата | Время (мин) | Журнал |
|---|------|----------|--------|------|-------------|--------|
| 7.N.1 | [Этап] | [Описание] | ⬜ | | — | |

## Канбан-доска
### 📋 Планируется / 🔍 Анализ / 🔧 Разработка / 🧪 Тестирование / ✅ Готово / ⏸️ Отложено

## Итог по времени
| Период | Задач завершено | Время (мин) | Время (ч:мин) |
|--------|----------------|-------------|----------------|
| 2026-05-16 | 8 | 45 | 0 ч 45 мин |

## Протокол управления (ключевые фразы)
| Фраза | Действие |
|-------|----------|
| «заводим задачу в дорожную карту» | Создать задачу |
| «дор карта 1» / «итого» | Сводная таблица |
| «дор карта 2» | Детализация всех задач |
| «статус задачи N» | Детализация задачи N |
| «обнови статус N.M» | Изменить статус этапа |
| «закрываем задачу N» | Пометить готово |
| «откладываем задачу N» | Отложить |
| «итого время» | Сводка времени (мин) |
| «затрачено N мин на задачу M» | Вписать время |
```

## Шаг 3. Настроить правила AI (.roo/rules)

Добавить в файл `.roo/rules` следующую секцию:

```markdown
## Roadmap & Task Management
- Roadmap file: [путь к ДорожнаяКарта.md]
- Change log: [путь к ЖурналИзменений.md]
- Numbering: N (task) → N.N (stage) → N.N.N (step)
- Key phrase «заводим задачу в дорожную карту» → Create new task
- Key phrase «дор карта 1» / «итого» → Show summary table
- Key phrase «дор карта 2» → Show all task details
- Key phrase «статус задачи N» → Show task N details
- Key phrase «обнови статус N.M» → Change stage status, recalculate %
- Key phrase «закрываем задачу N» → Mark as ✅ Done, request time spent
- Key phrase «откладываем задачу N» → Mark as ⏸️ Deferred
- Key phrase «итого время» → Show time summary in minutes
- Key phrase «затрачено N мин на задачу M» → Write time to task M
- Always update roadmap when completing stages
- Always update ЖурналИзменений.md for code changes
- Track time: record start/end date, time in MINUTES for each task and stage
- Time unit: minutes (integer). Format: «N мин» or «N ч M мин» if ≥ 60
- Maintain ⏸️ Deferred tasks list with reason
```

## Статусы и их обозначения

### Задачи (уровень 1)

| Символ | Статус | Когда ставить |
|--------|--------|---------------|
| 📋 | Планируется | Задача создана, ещё не начата |
| 🔍 | Анализ | Начат первый этап (анализ) |
| 🔧 | Разработка | Идёт написание кода |
| 🧪 | Тестирование | Код написан, ожидает проверки |
| 🔒 | Заблокировано | Ожидает внешнего фактора (1С клиент, API, требования) |
| ✅ | Готово | Все этапы завершены |
| ⏸️ | Отложено | Задача отложена |

### Этапы (уровни 2-3)

| Символ | Статус |
|--------|--------|
| ⬜ | Не начат |
| 🔵 | В работе |
| ✅ | Выполнен |
| 🧪 | Тестирование |
| ⚠️ | Проблема |
| 🔒 | Заблокирован (внешний фактор) |
| ⏭️ | Пропущен |

### Шкала приоритетов (колонка P)

| Приоритет | Описание | Когда делать |
|-----------|----------|--------------|
| **P1** | Критический | Первым делом, блокирует другую работу |
| **P2** | Высокий | В текущем спринте/итерации |
| **P2.5** | Средний | После P2, но до P3 |
| **P3** | Низкий | Когда есть время |

### Подробное описание каждой команды

#### Команда 1: `«заводим задачу в дорожную карту»`

**Назначение:** Создание новой задачи в дорожной карте.

**Синтаксис:**
```
заводим задачу в дорожную карту: [название], приоритет [P], этапы: [список]
```

**Примеры:**
```
заводим задачу в дорожную карту: Настроить CI/CD, приоритет P2, этапы:
1) Настроить GitLab Runner
2) Написать .gitlab-ci.yml
3) Протестировать пайплайн
```

**Что делает AI:**
1. Назначает следующий свободный номер задачи
2. Создаёт этапы с нумерацией N.1, N.2, N.3...
3. Добавляет задачу в сводную таблицу (уровень 1)
4. Добавляет в канбан-доску в секцию «📋 Планируется»
5. Проверяет лимит активных задач (максимум 5)
6. Если лимит превышен → автоматически откладывает с причиной

**Связанные разделы плана:** [Шаг 5](#шаг-5-создать-первую-задачу), [Конвенция: Лимит активных задач](#конвенция-лимит-активных-задач)

#### Команда 2: `«дор карта 1»` или `«итого»`

**Назначение:** Показать сводную таблицу всех задач (уровень 1).

**Синтаксис:**
```
дор карта 1
итого
```

**Что делает AI:**
1. Читает файл `ДорожнаяКарта.md`
2. Извлекает секцию «Сводная таблица задач (Уровень 1)»
3. Показывает таблицу как есть

**Результат:** Таблица вида:
```
| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
```

**Связанные разделы плана:** [Шаг 2 → Сводная таблица](#обязательные-секции)

---

#### Команда 3: `«дор карта 2»`

**Назначение:** Показать полную детализацию всех задач (уровни 2 и 3).

**Синтаксис:**
```
дор карта 2
```

**Что делает AI:**
1. Читает файл `ДорожнаяКарта.md`
2. Извлекает все секции детализации по задачам
3. Показывает все этапы и шаги для каждой задачи

**Результат:** Полная информация по всем задачам с их этапами.

**Связанные разделы плана:** [Шаг 2 → Детализация](#обязательные-секции)

---

#### Команда 4: `«статус задачи N»`

**Назначение:** Показать детальную информацию по одной задаче.

**Синтаксис:**
```
статус задачи 1
статус задачи 6
статус задачи 7
```

**Что делает AI:**
1. Читает файл `ДорожнаяКарта.md`
2. Находит секцию задачи N
3. Показывает все этапы с их статусами
4. Для N=7 показывает все отложенные задачи с причинами

**Пример результата:**
```
### 6.8. Сравнение цепочки процессов заказов Ozon ✅

| № | Этап | Описание | Статус | Дата | Время (мин) |
|---|------|----------|--------|------|-------------|
| 6.8.1 | Карта процессов Ozon (УОТ607) | Построена полная цепочка | ✅ | 2026-05-16 | 15 |
| 6.8.2 | Карта процессов Ozon (УОТКОРП) | Выявлены разрывы | ✅ | 2026-05-16 | 20 |
```

**Связанные разделы плана:** [Шаг 2 → Детализация](#обязательные-секции), [Конвенция: Отложенные задачи](#конвенция-отложенные-задачи)

---

#### Команда 5: `«обнови статус N.M»`

**Назначение:** Изменить статус конкретного этапа задачи.

**Синтаксис:**
```
обнови статус 2.1          → AI спросит новый статус
обнови статус 2.1 на ✅    → AI сразу установит статус
```

**Что делает AI:**
1. Находит этап N.M в дорожной карте
2. Меняет статус (⬜ → 🔵 → ✅ и т.д.)
3. Пересчитывает процент выполнения задачи N
4. Обновляет статус задачи в уровне 1 (если первый этап → 🔍, если последний → ✅)
5. Обновляет канбан-доску

**Связанные разделы плана:** [Статусы → Этапы](#этапы-уровни-2-3)

---

#### Команда 6: `«закрываем задачу N»`

**Назначение:** Пометить задачу как выполненную.

**Синтаксис:**
```
закрываем задачу 2
```

**Что делает AI:**
1. Устанавливает все оставшиеся этапы в ✅
2. Устанавливает % = 100%
3. Записывает дату завершения
4. **Запрашивает затраченное время** в минутах
5. Переносит задачу в секцию «6. ✅ Выполненные задачи»
6. Обновляет канбан-доску (переносит в ✅ Готово)
7. Обновляет «Итог по времени»

**Пример диалога:**
```
Пользователь: закрываем задачу 3
AI: Задача 3 закрыта ✅. Сколько минут затрачено?
Пользователь: 45
AI: Время зафиксировано: 45 мин. Задача перенесена в секцию 6.
```

**Связанные разделы плана:** [Конвенция: Учёт времени](#конвенция-учёт-времени-в-минутах)

---

#### Команда 7: `«откладываем задачу N»`

**Назначение:** Отложить задачу с указанием причины.

**Синтаксис:**
```
откладываем задачу 3
откладываем задачу 3, причина: жду ответ от клиента
```

**Что делает AI:**
1. Переносит задачу в секцию «7. ⏸️ Отложенные задачи»
2. Увеличивает счётчик в уровне 1
3. Записывает причину откладывания
4. Обновляет канбан-доску (переносит в ⏸️ Отложено)

**Связанные разделы плана:** [Конвенция: Отложенные задачи](#конвенция-отложенные-задачи)

---

#### Команда 8: `«итого время»`

**Назначение:** Показать сводку затраченного времени по всем задачам.

**Синтаксис:**
```
итого время
```

**Что делает AI:**
1. Читает все значения из колонки «Время (мин)» для всех задач
2. Суммирует общее время
3. Группирует по периодам (дням)
4. Конвертирует в удобный формат (мин → ч:мин при ≥ 60)

**Пример результата:**
```
| Период | Задач завершено | Время (мин) | Время (ч:мин) |
|--------|----------------|-------------|----------------|
| 2026-05-16 | 8 | 245 | 4 ч 5 мин |
| ИТОГО | 8 | 245 | 4 ч 5 мин |
```

## Версионирование планов

### Правила

1. **Текущая версия** всегда в файле `План_СистемаДорожныхКарт.md` (без суффикса версии)
2. **Архивная версия** — файл `План_СистемаДорожныхКарт_vN.md` (N = номер версии)
3. При значительных изменениях плана:
   - Скопировать текущий файл → `План_СистемаДорожныхКарт_vN.md`
   - Добавить пометку `⚠️ АРХИВ` в начало архивного файла
   - Обновить основной файл с увеличением версии
4. **Журнал версий** — таблица в начале файла с колонками: Версия, Дата, Сессия, Изменения
5. **Связь план↔карта**: при каждом изменении плана проверять соответствие с `ДорожнаяКарта.md`

### Когда создавать новую версию

| Событие | Действие |
|---------|----------|
| Добавлен новый статус/символ | Новая версия |
| Изменена структура таблиц | Новая версия |
| Добавлена новая ключевая фраза | Новая версия |
| Изменён формат учёта времени | Новая версия |
| Добавлено руководство по командам | Новая версия |
| Исправление опечатки | Без новой версии |
| Обновление примеров | Без новой версии |

### Соответствие план ↔ дорожная карта

При рассинхронизации (дорожная карта эволюционировала быстрее плана):
1. Зафиксировать расхождение в журнале версий
2. Обновить план до соответствия карте
3. Создать новую версию плана
