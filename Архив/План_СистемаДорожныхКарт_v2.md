## Обзор системы

Система состоит из 3 компонентов:

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **Дорожная карта** | `ДорожнаяКарта.md` | Файл с задачами, этапами, статусами, временем |
| **Правила AI** | `.roo/rules` | Инструкции для AI-ассистента (ключевые фразы) |
| **Журнал изменений** | `ЖурналИзменений.md` | История всех изменений с детализацией |

## Шаг 1. Создать структуру каталогов

```
проект/
├── .roo/
│   ├── rules          ← Правила для AI (обязательный)
│   └── rules-code     ← Правила для режима Code (опционально)
├── docs/              ← Документация проекта (любое имя)
│   ├── ДорожнаяКарта.md
│   └── ЖурналИзменений.md
└── CONTEXT.md         ← Быстрый контекст проекта
```

## Шаг 2. Создать файл дорожной карты

Создать файл `docs/ДорожнаяКарта.md` (или в корне проекта) со следующей структурой:

### Обязательные секции:

### Конвенция: Лимит активных задач

- **Максимум 5 активных задач** (не отложенных) в дорожной карте
- При создании 6-й задачи без явного указания приоритета → **автоматически отложить** (перенести в секцию 7)
- Причина автооткладывания: «Превышен лимит активных задач (максимум 5)»
- Если одна из 5 задач закрыта (✅) → можно вытащить задачу из отложенных

### Конвенция: Отложенные задачи

- В **уровне 1** (дор карта 1) отложенные задачи отображаются **одной строкой**: `| **7** | ⏸️ Отложенные задачи (N шт.) |`
- В **уровне 2** (дор карта 2 / «статус задачи 7») — список всех отложенных задач с нумерацией `7.1`, `7.2`, `7.3`...
- В **уровне 3** — этапы каждой отложенной задачи: `7.1.1`, `7.1.2`...
- При команде «статус задачи 7» AI выдаёт все отложенные задачи с причинами и приоритетами
- При добавлении новой отложенной задачи: увеличить счётчик в уровне 1, добавить строку 7.N в уровень 2

```markdown
# Дорожная карта проекта [ИМЯ]

## Структура нумерации
| Уровень | Формат | Пример | Описание |
|---------|--------|--------|----------|
| 1 (задача) | N | 1 | Крупная задача |
| 2 (этап) | N.N | 1.1 | Стадия выполнения |
| 3 (шаг) | N.N.N | 1.1.1 | Конкретное действие |

## Сводная таблица задач (Уровень 1)
| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время |
|---|--------|---|-----------|---|--------|--------|-------|-------|
| 1 | [Название] | P1 | ⭐ | 0% | 📋 | — | — | — |

## Детализация по задачам (Уровни 2 и 3)
### 1. [Название задачи] 📋
| № | Этап | Описание | Статус | Дата | Время | Журнал |
|---|------|----------|--------|------|-------|--------|
| 1.1 | [Этап] | [Описание] | ⬜ | | | |

## 7. ⏸️ Отложенные задачи
> В уровне 1 — одна строка: `| **7** | ⏸️ Отложенные задачи (N шт.) | — | — | — | ⏸️ | — | — | — |`

| № | Задача | Причина | P | Дата |
|---|--------|---------|---|------|

### 7.N [Название отложенной задачи] ⏸️
| № | Этап | Описание | Статус | Дата | Время | Журнал |
|---|------|----------|--------|------|-------|--------|
| 7.N.1 | [Этап] | [Описание] | ⬜ | | | |

## Канбан-доска
### 📋 Планируется / 🔍 Анализ / 🔧 Разработка / 🧪 Тестирование / ✅ Готово / ⏸️ Отложено

## Итог по времени
| Период | Задач завершено | Время |
|--------|----------------|-------|

## Протокол управления (ключевые фразы)
| Фраза | Действие |
|-------|----------|
| «заводим задачу в дорожную карту» | Создать задачу |
| «дор карта 1» / «итого» | Сводная таблица |
| «статус задачи N» | Детализация |
| «обнови статус N.M» | Изменить статус |
| «закрываем задачу N» | Пометить готово |
| «откладываем задачу N» | Отложить |
| «итого время» | Сводка времени |
```

## Шаг 3. Настроить правила AI (.roo/rules)

Добавить в файл `.roo/rules` следующую секцию:

```markdown
## Roadmap & Task Management
- Roadmap file: [путь к ДорожнаяКарта.md]
- Change log: [путь к ЖурналИзменений.md]
- Numbering: N (task) → N.N (stage) → N.N.N (step)
- Key phrase «заводим задачу в дорожную карту» → Create new task
- Key phrase «статус задачи N» → Show task N details
- Key phrase «обнови статус N.M» → Change stage status, recalculate %
- Key phrase «закрываем задачу N» → Mark as ✅ Done
- Key phrase «откладываем задачу N» → Mark as ⏸️ Deferred
- Key phrase «итого» / «сводка» → Show summary table
- Key phrase «итого время» → Show time summary
- Always update roadmap when completing stages
- Always update ЖурналИзменений.md for code changes
- Track time: record start/end date for each task and stage
- Maintain ⏸️ Deferred tasks list with reason
```

## Шаг 5. Создать первую задачу

Сказать AI-ассистенту:

> **«заводим задачу в дорожную карту: [название задачи], приоритет P1, этапы: [список этапов]»**

AI автоматически:
1. Назначит следующий свободный номер
2. Создаст этапы с нумерацией N.1, N.2, ...
3. Добавит задачу в сводную таблицу
4. Добавит в канбан «Планируется»

## Статусы и их обозначения

### Задачи (уровень 1)

| Символ | Статус | Когда ставить |
|--------|--------|---------------|
| 📋 | Планируется | Задача создана, ещё не начата |
| 🔍 | Анализ | Начат первый этап (анализ) |
| 🔧 | Разработка | Идёт написание кода |
| 🧪 | Тестирование | Код написан, ожидает проверки |
| 🔒 | Заблокировано | Ожидает внешнего фактора (1С клиент, API, требования) |
| ✅ | Готово | Все этапы завершены |
| ⏸️ | Отложено | Задача отложена |

### Этапы (уровни 2-3)

| Символ | Статус |
|--------|--------|
| ⬜ | Не начат |
| 🔵 | В работе |
| ✅ | Выполнен |
| 🧪 | Тестирование |
| ⚠️ | Проблема |
| 🔒 | Заблокирован (внешний фактор) |
| ⏭️ | Пропущен |

### Шкала приоритетов (колонка P)

| Приоритет | Описание | Когда делать |
|-----------|----------|--------------|
| **P1** | Критический | Первым делом, блокирует другую работу |
| **P2** | Высокий | В текущем спринте/итерации |
| **P2.5** | Средний | После P2, но до P3 |
| **P3** | Низкий | Когда есть время |

## Примеры использования

### Создать задачу
> «заводим задачу в дорожную карту: Настроить CI/CD, приоритет P2, этапы: 1) Настроить GitLab Runner 2) Написать .gitlab-ci.yml 3) Протестировать пайплайн»

### Посмотреть статус
> «дор карта 1» — сводная таблица
> «статус задачи 2» — детализация по этапам

### Обновить прогресс
> «обнови статус 2.1» — AI изменит ⬜→🔵 или 🔵→✅

### Завершить
> «закрываем задачу 2» — AI запишет дату, время, перенесёт в ✅ Готово

### Подвести итоги
> «итого время» — AI покажет сколько времени потрачено на каждую задачу

## Версионирование планов

### Правила

1. **Текущая версия** всегда в файле `План_СистемаДорожныхКарт.md` (без суффикса версии)
2. **Архивная версия** — файл `План_СистемаДорожныхКарт_vN.md` (N = номер версии)
3. При значительных изменениях плана:
   - Скопировать текущий файл → `План_СистемаДорожныхКарт_vN.md`
   - Добавить пометку `⚠️ АРХИВ` в начало архивного файла
   - Обновить основной файл с увеличением версии
4. **Журнал версий** — таблица в начале файла с колонками: Версия, Дата, Сессия, Изменения
5. **Связь план↔карта**: при каждом изменении плана проверять соответствие с `ДорожнаяКарта.md`

### Когда создавать новую версию

| Событие | Действие |
|---------|----------|
| Добавлен новый статус/символ | Новая версия |
| Изменена структура таблиц | Новая версия |
| Добавлена новая ключевая фраза | Новая версия |
| Исправление опечатки | Без новой версии |
| Обновление примеров | Без новой версии |

### Соответствие план ↔ дорожная карта

При рассинхронизации (дорожная карта эволюционировала быстрее плана):
1. Зафиксировать расхождение в журнале версий
2. Обновить план до соответствия карте
3. Создать новую версию плана
