2. Жизненный цикл тестов

Фаза 1: Открытие формы → Загрузка модулей

Пользователь открывает обработку "ЮТЮнитТесты"
↓ Форма.ПриСозданииНаСервере() → Загрузка списка тестовых модулей
↓ ЮТЧитательСлужебныйКлиент.ИсполняемыеСценарии() — перебор всех модулей
↓ Для каждого модуля: проверка наличия метода ИсполняемыеСценарии()

Фаза 2: Построение дерева тестов (Discovery)

ЮТЧитательСлужебный.ИсполняемыеСценарииМодуля()
↓ Вызов: Модуль.ИсполняемыеСценарии() ← точка входа
↓ Внутри ИсполняемыеСценарии():
  ЮТТесты.ДобавитьТестовыйНабор("Имя") → создаёт suite
  .ДобавитьТест("Метод", "Представление") → добавляет тест
  .ДобавитьКлиентскийТест(...) → клиентский контекст
  .ДобавитьСерверныйТест(...) → серверный контекст
  .Настроить(ЮТАлюр.Epic(...)) → теги Allure
↓ Результат: ОписаниеТестовогоМодуля (наборы → тесты → параметры)

Фаза 3: Запуск тестов (Execution)

Пользователь нажимает "Запустить"
↓ ЮТИсполнительСлужебныйКлиент.ВыполнитьТестовыеМодули()
↓ Для каждого модуля → набора → теста:
  ЮТМетодыСлужебный.ВыполнитьМетодОбъектаСПерехватомОшибки()
↓ Выполнить("Объект.МетодТеста(Параметры)") ← прямой вызов
↓ Запись результата → отображение в UI + Allure XML

4. Типы тестовых модулей

4.1 Интеграционные тесты (API)

Модуль: CG_Online_UT_ИнтеграцииССервисами_ЮнитТестыКлиентСервер (1087 строк)

Тестирование API-методов Ozon/WB/Яндекс/Авито через макеты Мокито.

// Пример регистрации
ЮТТесты
    .ДобавитьТестовыйНабор("Интеграция (Загрузка товаров)")
    .ДобавитьСерверныйТест("ЗагрузитьКатегорииТоваров", "Загрузить категории", Сервер)
    .ДобавитьСерверныйТест("ЗагрузитьТовары", "Загрузить товары", Сервер)

4.2 Дымовые тесты открытия форм (ДТ)

Модули: ДТОткрытиеФорм, CG_Online_UT_ДТ_ДокументыКлиент, CG_Online_UT_ДТ_СправочникиКлиент и др.

Автоматическая генерация тестов из метаданных конфигурации. Дерево тестов строится динамически.

// ДТОткрытиеФорм.ИсполняемыеСценарии()
ЮТТесты.Настроить(ЮТАлюр.Epic("Дымовые тесты"));
ДеревоТестов = ДТОткрытиеФормСлужебныйВызовСервера.ТипыТестов();
// → Перебор коллекций → объектов → типов тестов

5. Система макетов (Мокито)

МодульНазначение
МокитоОбучениеНастройка ожидаемых вызовов и ответов
МокитоПерехватПерехват вызовов API в runtime
МокитоПроверкиПроверка факта вызова и параметров
CG_Online_UT_МокитоСпецифичные для КОРПтест настройки

Паттерн использования

// 1. Настроить макет
МокитоОбучение.Настроить("CG_Online_Ozon_ИнтеграцияСОзон", "ВызватьAPI")
    .Когда("v3/product/list")
    .Вернуть(CG_Online_UT_РаботаСТестовымиДанными.Макет("CG_Online_Ozon_JSON_ДанныеОТоварах_Краткий"));

// 2. Выполнить тестируемый код
Результат = ТестируемыйМодуль.ЗагрузитьТовары();

// 3. Проверить результат
ЮТест.ОжидаетЧто(Результат).Заполнено();
МокитоПроверки.Проверить("...", "ВызватьAPI").Вызывался(1);
