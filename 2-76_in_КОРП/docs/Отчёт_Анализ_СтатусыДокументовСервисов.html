8. Выводы

Ключевые различия архитектуры

1. Модульная архитектура КОРП
УОТКОРП использует развитую модульную архитектуру с 8 CommonModule'ями вместо прямого вызова из ManagerModule. Это позволяет поддерживать разные модели работы (ИСП, ИССОХ, ИСТП, ТП) через проверку подсистем.
2. Двойной источник данных в КОРП
В УОТКОРП статусы загружаются из двух источников:
• DataProcessor'ы сервисов — при наличии настроек авторизации (как в УОТ609)
• Шаблоны на справочнике — при первом запуске / отсутствии настроек (4 макета по моделям работы)
3. Критическое отличие: имя справочника
УОТ609: CG_Online_IWS_СтатусыДокументовСервисов
УОТКОРП: CG_Online_СтатусыДокументовСервисов
При миграции/синхронизации требуется маппинг имён!
4. Расширенный функционал КОРП
УОТКОРП добавляет: версионирование, блокирование реквизитов, синхронизацию данных, создание на основании, печать, панель администрирования с 4+ формами управления статусами.
