🔍 Анализ: «Недостаточно данных» — Дымовые тесты

Документ: CG_Online_ВводНачальныхОстатковТоваровВСервисе  |  Расширение: КОРПтест  |  Дата: 2026-05-27

⚠️ Причина

Дымовые тесты «Открыть форму существующего объекта» не создают документы — они ищут уже существующие в базе данных по фильтру.

Тест пропускается (ЮТест.Пропустить("Недостаточно данных")), если ни один документ не проходит условия отбора.

Цепочка вызовов

1
CG_Online_UT_ДТ_ДокументыКлиент.ОткрытьФормуСуществующегоОбъекта()
ПараметрыОтбора = ДТС_Настройки.ПараметрыОтбораДокумента(Параметры)
2
ДТС_Настройки.ПараметрыОтбораДокумента()
Формирует фильтр: ПометкаУдаления = Ложь, Проведен = Ложь, Дата >= &Дата
3
ДТС_Тесты.ОткрытьФормуОбъекта()
Ссылка = ДТС_СлучайныеДанныеСервер.СлучайнаяСсылка(Тип, Имя, ПараметрыОтбора)
4
ДТС_СлучайныеДанныеСервер.СлучайнаяСсылка()
Выполняет запрос к БД с фильтром из шага 2
5
Результат пустой → Ссылка = Неопределено
ЮТест.Пропустить("Недостаточно данных")

Фильтр отбора документов

Условие Значение Результат для данного документа
ПометкаУдаления = Ложь Не удалён ✅ Обычно проходит
Проведен = Ложь Не проведён ❌ Документ обычно проведён
Дата >= &Дата Дата ≥ ТекущаяДата − 7 дней ❌ Создан давно

Почему документ не находится

  • 📋 Документ «Ввод начальных остатков товаров в сервисе» — это документ начальной настройки. Создаётся однократно при внедрении интеграции с сервисом.
  • ✅ Сразу проводится после создания — поэтому условие Проведен = Ложь не выполняется.
  • 📅 Дата создания — давно — не попадает в фильтр Дата >= ТекущаяДата() − 7 дней.

🛠️ Предлагаемые решения

✅ Вариант 1 (рекомендуемый) Рекомендуется

Расширить диапазон дат в ОтборДата() — вместо недели использовать начало года.

КОРПтест/CommonModules/ДТС_Настройки/Ext/Module.bsl : 652

Было:
Функция ОтборДата()
    Возврат НачалоДня(ТекущаяДата()) - (60 * 60 * 24 * 7); // Неделя
КонецФункции
Стало:
Функция ОтборДата()
    Возврат НачалоГода(ТекущаяДата()); // Начало года
КонецФункции
👍 Плюсы
  • Решает проблему для всех документов
  • Простое изменение — одна строка
  • Не требует поддержки списка исключений
⚠️ Минусы
  • Запрос может вернуть больше записей
  • Ограничение КоличествоЗаписей = 1000 компенсирует
🔧 Вариант 2 Альтернатива

Убрать фильтр по дате для непроведённых документов в ПараметрыОтбораДокумента().

КОРПтест/CommonModules/ДТС_Настройки/Ext/Module.bsl : 51

Функция ПараметрыОтбораДокумента(Параметры) Экспорт
    Условия = Новый Массив;
    Условия.Добавить("ПометкаУдаления = Ложь");
    Условия.Добавить("Проведен = Ложь");
    // Убрано: Условия.Добавить("Дата >= &Дата");
    Возврат ПараметрыОтбора(Условия);  // Без ЗначенияПараметров
КонецФункции
👍 Плюсы
  • Наиболее точное решение
  • Непроведённых документов обычно немного
⚠️ Минусы
  • Может найти очень старые документы
  • Изменяет логику для всех типов документов
📝 Вариант 3 Точечный

Добавить в список исключений в НастройкиИсключенийДокументов().

КОРПтест/CommonModules/ДТС_Настройки/Ext/Module.bsl : 377

НастройкиИсключений.Добавить("Документ.CG_Online_ВводНачальныхОстатковТоваровВСервисе.ОткрытьФормуСуществующегоОбъекта");
НастройкиИсключений.Добавить("Документ.CG_Online_ВводНачальныхОстатковТоваровВСервисе.ЗаписатьСуществующийОбъект");
НастройкиИсключений.Добавить("Документ.CG_Online_ВводНачальныхОстатковТоваровВСервисе.ОткрытьФормуСуществующегоПроведенногоОбъекта");
НастройкиИсключений.Добавить("Документ.CG_Online_ВводНачальныхОстатковТоваровВСервисе.ПровестиСуществующийОбъект");
👍 Плюсы
  • Точечное решение
  • Не влияет на другие документы
⚠️ Минусы
  • Тесты для этого документа вообще не выполняются
  • Нужно поддерживать список исключений

📊 Какие тесты пропускаются

Для Документ.CG_Online_ВводНачальныхОстатковТоваровВСервисе из НаборТестовДокумента() генерируются 7 тестов:

# Тест Нужны данные Результат
1 ОткрытьФормуСписка Нет ✅ Выполняется
2 ОткрытьФормуВыбора Нет ✅ Выполняется
3 ОткрытьФормуОбъекта (новый) Нет ✅ Выполняется
4 ОткрытьФормуСуществующегоОбъекта Да ⚠️ Пропуск
5 ЗаписатьСуществующийОбъект Да ⚠️ Пропуск
6 ОткрытьФормуСуществующегоПроведенногоОбъекта Да ⚠️ Пропуск
7 ПровестиСуществующийОбъект Да ⚠️ Пропуск
