# Дорожная карта проектов

Система дорожных карт для отслеживания задач по нескольким проектам. Каждый проект имеет свои задачи с этапами, приоритетами и прогрессом.

## Структура (шаблон)

```
ДорожнаяКарта/
├── ДорожнаяКарта.md            ← Единая сводка по всем проектам
├── Проект1/                    ← Папка проекта (имя произвольное)
│   ├── ДорожнаяКарта.md         Детальная карта задач
│   ├── tasks.json               Этапы задач (машинная версия)
│   └── docs/                    Отчёты и аналитика (HTML/MD)
├── Проект2/                    ← Второй проект
│   ├── tasks.json
│   └── docs/
├── Проект3/                    ← Третий проект
│   ├── tasks.json
│   └── docs/
├── ПравилаДорожнойКарты.md      Правила ведения
└── План_СистемаДорожныхКарт_v*.md  Версионные снимки плана
```

> `Проект1`, `Проект2`, `Проект3` — пример. Вы можете называть папки как угодно (например `Backend`, `Mobile`, `Analytics`).

## Команды (ключевые фразы)

Система управляется через AI-ассистента (Roo Code / Claude CLI) ключевыми фразами в чате:

| Команда | Действие |
|---------|----------|
| `др` / `сводка` / `дорожная карта` | Показать полную таблицу всех проектов |
| `проект N` | Переключиться на проект N |
| `статус задачи N` | Показать детализацию задачи с этапами |
| `обнови статус N.M` | Отметить этап выполненным |
| `закрываем задачу N` | Пометить задачу как готово |
| `откладываем задачу N` | Перенести в отложенные |
| `затрачено N мин на задачу M` | Записать время |
| `итого время` | Сводка затраченного времени |
| `заводим задачу в дорожную карту` | Создать новую задачу |
| `верни задачу N в активные` | Вернуть из отложенных в активные |

## Как пользоваться

### Быстрый старт
1. Откройте `ДорожнаяКарта.md` — это сводная таблица по всем проектам
2. Скажите `др` — AI покажет актуальный статус
3. Скажите `проект 1` — переключитесь на нужный проект
4. Скажите `статус задачи 3` — посмотрите детали этапов

### Пример работы

```
→ др
← (показывает таблицу всех проектов и задач)
→ проект 2
← (переключились на проект 2, видны его задачи)
→ обнови статус 2.1
← (этап 2.1 отмечен как выполненный)
→ закрываем задачу 3
← (задача 3 перенесена в выполненные)
```

### Структура задачи
Каждая задача состоит из этапов (уровень 2) и шагов (уровень 3):
- `1.0` — задача "Внедрение заказов"
- `1.1` — этап "Настройка коннектора"
- `1.1.1` — шаг "Создать HTTP-соединение" (опционально)

### Имена проектов
Папки проектов называются произвольно. Главное — чтобы имя совпадало в `ДорожнаяКарта.md` (сводка) и в имени папки проекта.

## Запуск

1. Скопировать папку `ДорожнаяКарта/` в корень рабочей области
2. Открыть `ДорожнаяКарта.md` — это главный файл сводки
3. Для работы с командами — использовать AI-ассистента с доступом к файлам дорожной карты

### Запуск по плану

В файлах `План_СистемаДорожныхКарт_v*.md` содержится детальное описание системы:
- Правила ведения и нумерации задач
- Структура этапов и уровней
- Протокол управления ключевыми фразами
- Схема переноса задач между статусами

Перед началом работы рекомендуется прочитать актуальную версию плана — последнюю по номеру `v*` (наибольшее число после `v`). В ней описаны все правила и соглашения.

### Примеры команд AI-ассистенту

Дайте ассистенту одну из следующих команд, чтобы он создал структуру дорожной карты по плану:

```
Создай дорожную карту согласно плану План_СистемаДорожныхКарт_v*.md
Разверни систему дорожных карт по плану План_СистемаДорожныхКарт_v*.md
Настрой дорожную карту по плану План_СистемаДорожныхКарт_v*.md
Инициализируй дорожную карту из плана План_СистемаДорожныхКарт_v*.md
```

Ассистент прочитает указанный файл плана и создаст структуру папок, файлов и задач в соответствии с ним.

## Содержание git

В репозиторий попадают только версионные снимки плана (`План_СистемаДорожныхКарт_v*.md`). Текущие рабочие файлы (`ДорожнаяКарта.md`, `tasks.json`, отчеты) хранятся локально.
