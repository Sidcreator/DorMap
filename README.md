# Дорожная карта проектов

Система дорожных карт для отслеживания задач по нескольким проектам. Каждый проект имеет свои задачи с этапами, приоритетами и прогрессом.

## Структура

```
ДорожнаяКарта/
├── ДорожнаяКарта.md          ← Единая сводка по всем проектам
├── Проект1/                  ← Проект 1 (папка с произвольным именем)
│   ├── ДорожнаяКарта.md       Детальная карта задач
│   ├── tasks.json             Этапы задач (машинная версия)
│   └── docs/                  HTML-отчёты и аналитика
├── Проект2/                  ← Проект 2
│   ├── tasks.json             Все задачи с этапами
│   └── docs/                  Отчёты
├── Проект3/                  ← Проект 3
│   ├── tasks.json             Задачи проекта
│   └── docs/                  Отчёты
├── ПравилаДорожнойКарты.md    Правила ведения
└── План_СистемаДорожныхКарт_v*.md   Версионные снимки плана
```

## Как пользоваться

### Просмотр статуса задач
Откройте `ДорожнаяКарта.md` — сводная таблица по всем проектам с приоритетами, прогрессом и статусами.

### Управление задачами
Все изменения ведутся в файлах:
- `ДорожнаяКарта.md` — сводка (обновлять при изменении любой задачи)
- `{проект}/tasks.json` — детальные этапы и прогресс
- `{проект}/ДорожнаяКарта.md` — детальная карта (опционально)

### Имена проектов
Папки проектов называются произвольно. Главное — чтобы имя совпадало в `ДорожнаяКарта.md` (сводка) и в имени папки проекта.

## Запуск (однократно)

1. Скопировать папку `ДорожнаяКарта/` в корень рабочей области
2. Открыть `ДорожнаяКарта.md` в любом Markdown-редакторе (VS Code, Obsidian, Notion)
3. Для навигации по проектам — использовать ссылки в сводной таблице

## Содержание git

В репозиторий попадают только версионные снимки плана (`План_СистемаДорожныхКарт_v*.md`). Текущие рабочие файлы (`ДорожнаяКарта.md`, `tasks.json`, отчеты) хранятся локально.
