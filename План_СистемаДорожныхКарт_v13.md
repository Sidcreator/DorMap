# План системы дорожных карт v13

**Дата**: 2026-06-28
**Назначение**: Полный план для развёртывания системы дорожных карт с нуля. Самодостаточный документ — AI, получив этот файл, создаёт все необходимые файлы.

---

## 1. Архитектура

### 1.1. Единый источник проектов: `projects.json`

Список всех проектов в корне `ДорожнаяКарта/`. Формат:

```json
{
  "version": 1,
  "defaultProject": "ИмяПроектаПоУмолчанию",
  "projects": [
    {
      "id": 1,
      "name": "КороткоеИмя",
      "fullName": "N. ПолноеИмя",
      "path": "D:\\полный\\путь\\к\\проекту",
      "roadmapDir": "N-имя-папки",
      "roadmapFile": "N-имя-папки/ДорожнаяКарта.md",
      "commands": ["проект N", "проект N/Имя", "прN"]
    }
  ]
}
```

Поля:
- `id` — номер проекта (1, 2, 3...)
- `name` — короткое имя для команд
- `fullName` — полное имя с номером для отображения
- `path` — полный путь к папке проекта
- `roadmapDir` — имя подпапки в `ДорожнаяКарта/`
- `commands` — список команд для переключения проекта

**Правило:** все изменения списка проектов — только в `projects.json`.

### 1.2. Единый источник задач: `tasks.json`

Каждый проект имеет `tasks.json` в своей папке `ДорожнаяКарта/N-проект/`. Структура задачи:

```json
{
  "id": 1,
  "name": "Название задачи",
  "priority": "P2",
  "complexity": 2,
  "progress": 0,
  "section": "active",
  "start": "2026-06-25",
  "end": null,
  "time_min": null,
  "stages": [
    { "id": "1.1", "name": "Этап", "desc": "Описание", "status": "⬜", "date": null, "report": null }
  ]
}
```

Поля:
- `section` — `"active"` / `"completed"` / `"postponed"`
- `stages[].status` — `⬜` (не начат), `🔵` (в работе), `✅` (выполнен)
- `stages[].report` — имя HTML-файла отчёта (опционально)
- Для `completed` задач: `start`, `end`, `time_min`
- Для `postponed` задач: `reason`

**Workflow:**
1. Все изменения задач вносятся только в `tasks.json`
2. После изменения AI перегенерирует `.md` файл проекта
3. Команда `др` строит сводку из `tasks.json` (не из `.md`)

### 1.3. Представления: `.md` файлы

Генерируются из `tasks.json`. Редактирование только через `tasks.json`.

### 1.4. Структура папок

```
ДорожнаяКарта/                          ← корень
├── projects.json                       ← единый список проектов
├── ДорожнаяКарта.md                    ← единая сводка (выбор проекта + таблица)
├── ПравилаДорожнойКарты.md             ← правила ведения
├── README.md                           ← быстрый старт
├── План_СистемаДорожныхКарт_v*.md      ← версионные планы
├── Новое в ДорожнойКарте_v*.md         ← описание изменений
├── Инструкция_Развертывание_без_AI_v*.md ← ручное развёртывание
├── 1-76/                               ← проект
│   ├── ДорожнаяКарта.md                ← детализация задач
│   ├── tasks.json                      ← задачи (источник истины)
│   └── отчеты/                         ← HTML-отчёты
├── 2-76_in_КОРП/                       ← проект
│   ├── ДорожнаяКарта.md
│   ├── tasks.json
│   └── отчеты/

│   ├── tasks.json
│   └── отчеты/
```

## 2. Структура нумерации

| Уровень | Формат | Пример | Описание |
|---------|--------|--------|----------|
| 1 (задача) | `N` | `1` | Крупная задача |
| 2 (этап) | `N.N` | `1.1` | Стадия выполнения |
| 3 (шаг) | `N.N.N` | `1.2.1` | Конкретное действие |

Правила нумерации:
- Номера задач — последовательно, следующий свободный
- Активные задачи: 1, 2, 3... (максимум 5)
- Выполненные: группируются под задачей `N` (напр. 7.1, 7.2)
- Отложенные: группируются под задачей `M` (напр. 8.1, 8.2)
- При переносе из выполненных/отложенных в активные — новый активный номер

## 3. Статусы

| Символ | Статус | Описание |
|--------|--------|----------|
| 📋 | Планируется | В очереди, не начата |
| 🔍 | Анализ | Начат анализ |

| 🧪 | Тестирование | Ожидает проверки |
| 🔒 | Заблокировано | Внешний фактор |
| ✅ | Готово | Все этапы завершены |
| ⏸️ | Отложено | Отложена по приоритету |

| Символ | Статус |
|--------|--------|
| ⬜ | Не начат |
| 🔵 | В работе |

## 4. Команды управления

### 4.1. Просмотр

| Команда | Действие |
|---------|----------|
| `др` / `др1` / `сводка` / `дорожная карта` / `итого` / `полная др` / `подробная` | Сводка **всех проектов**: активные + ✅ + ⏸️ + итого |
| `др2` / `дор карта 2` | Детализация текущего проекта (уровни 2-3) |
| `только активные` | Только активные задачи (без ✅ и ⏸️) |
| `проект N` / `прN` / `пр N` | Переключиться на проект N |
| `стN` / `статус N` | Статус проекта N (без переключения) |
| `стз N` / `статус задачи N` | Детализация задачи N в текущем проекте |

### 4.2. Управление задачами

| Команда | Действие |
|---------|----------|
| `заводим задачу в дорожную карту` | Создать задачу. Проверить лимит 5 активных |
| `обнови статус N.M` | Изменить статус этапа, пересчитать % |
| `закрываем задачу N` | ✅ Пометить готово, запросить время |
| `откладываем задачу N` | ⏸️ Перенести в отложенные |
| `верни задачу N в активные` / `перенеси задачу N в активные` | Вернуть из выполненных/отложенных в активные с новым номером |

### 4.3. Учёт времени

| Команда | Действие |
|---------|----------|
| `затрачено N мин на задачу M` | Вписать время |
| `итого время` | Сводка времени (мин / ч:мин) |

### 4.4. Отчёты и правила

| Команда | Действие |
|---------|----------|
| `отк отчет` / `отк от` / `отк отчет текущий` / `отк от тек` | Открыть последний HTML-отчёт |
| `отк отчет ИмяФайла` | Открыть отчёт по имени из `отчеты/` |
| `отк отчет N.N` | Открыть отчёт, привязанный к этапу N.N |
| `правила дорожной карты` | Показать правила |
| `обновить правила дорожной карты` | Обновить все файлы системы, создать новую версию плана |
| `синхронизация` | Синхронизировать правила между всеми проектами |

## 5. Правила отображения

При команде `др` — всегда показывать **все 3 раздела** для каждого проекта:

| Раздел | Что показывать | Формат |
|--------|---------------|--------|
| Активные | Все задачи со статусами 📋🔍🔧🧪 | №, Задача, P, Сложность, %, Статус |
| Выполненные | Все задачи ✅ | №, Задача, P, Дата |
| Отложенные | Все задачи ⏸️ | №, Задача, P, Причина |
| Итого | Сводка по проекту | Активных / ✅ / ⏸️ |

**Правило единообразия:** для всех проектов — полный список задач в каждом разделе, без сокращений.

## 6. Конвенции

### 6.1. Лимит активных задач
- Максимум **5 активных задач** одновременно
- При создании 6-й — авто-отложить с причиной

### 6.2. Учёт времени
- Единица: минуты (целое число)
- При старте: `—` (прочерк)
- При завершении: конкретное число
- При промежуточном: `~N мин`
- Итог: `N мин` или `N ч M мин` при N ≥ 60

### 6.3. Именование файлов
- Вспомогательные файлы — с суффиксом `_vN` (напр. `Инструкция_Развертывание_без_AI_v13.md`)
- Основные рабочие файлы — без суффикса: `ДорожнаяКарта.md`, `ПравилаДорожнойКарты.md`, `README.md`, `projects.json`, `tasks.json`

### 6.4. План версии — полное описание
`План_СистемаДорожныхКарт_vN.md` — самодостаточный документ для развёртывания с нуля. Должен содержать **полную спецификацию**: архитектуру, все правила, команды, конвенции, форматы, лимиты. Не только изменения.

## 7. Формат таблиц

### Таблица этапов (уровень 2)

```
| № | Этап | Описание | Статус | Дата | Время (мин) | Отчёт | Журнал |
```

- Колонка **Отчёт** — ссылка на HTML-файл в `отчеты/`
- Формат ссылки: `[Отчет_N.N_...](отчеты/Отчет_N.N_...html)`
- Если отчёта нет — `—`

### Таблица отложенных (уровень 1)

```
| № | Задача | Причина | P | Дата |
```

## 8. Формат HTML-отчёта

Каждый HTML-отчёт содержит:

1. **Шапка `<div class="meta">`**:
   - Метод / endpoint
   - Модуль / файл (если применимо)
   - Строка в коде (если применимо)
   - **Задача ДК** — номер задачи и этапа
   - **Проект** — номер и имя проекта
   - Дата создания

2. **Подвал**: `Проект N/Имя — Дорожная карта. Отчёт привязан к задаче N.N.`

3. **CSS-классы**:
   - `.issue` (красный) — проблемы
   - `.ok` (зелёный) — успешные проверки
   - `.warn` (оранжевый) — предупреждения

4. **Имя файла**: `Отчет_N.N_КраткоеОписание.html`
   - Пример: `Отчет_4.1_Анализ_v1_analytics_stocks.html`

## 11. Развёртывание (инструкция для AI)

При получении команды «Разверни систему дорожных карт по плану» AI выполняет:

### 11.1. Создать структуру папок

```
ДорожнаяКарта/
├── N-проект1/
│   └── отчеты/
├── N-проект2/
│   └── отчеты/
...
```

Где `N-проект` — для каждого проекта из запроса пользователя. Имена папок по шаблону `N-КороткоеИмя`.

### 11.2. Создать projects.json

По шаблону из раздела 1.1. Запросить у пользователя список проектов (id, name, fullName, path). Если пользователь не указал — создать с одним тестовым проектом.

### 11.3. Создать ПравилаДорожнойКарты.md

Включить полное содержимое разделов 2-10 данного плана (структура нумерации, статусы, команды, конвенции, форматы, лимиты).

### 11.4. Создать README.md

Содержит:
- Краткое описание системы
- Таблицу команд (раздел 4 данного плана)
- Структуру папок (раздел 1.4)
- Быстрый старт: команда `др` → просмотр → управление

### 11.5. Создать ДорожнаяКарта.md (единая сводка)

Содержит:
- Выбор проекта (таблица из projects.json)
- Сводная таблица задач (раздел 5)
- Протокол отображения (раздел 5)
- Итого по проектам

### 11.6. Для каждого проекта создать:

1. **`tasks.json`** — пустой или с одним примером задачи
2. **`ДорожнаяКарта.md`** — сводная таблица задач + детализация этапов + протокол управления
3. **Пустую папку `отчеты/`**

### 11.7. Создать Инструкция_Развертывание_без_AI_vN.md

Содержит пошаговое руководство для человека (копирование файлов, настройка projects.json, инициализация, проверка).

### 11.8. Финальная проверка

Выполнить команду `др` — AI должен показать сводную таблицу всех проектов с разделами: активные + выполненные + отложенные.
