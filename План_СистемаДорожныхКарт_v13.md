## 1. Архитектура

### 1.1. Единый источник проектов: `projects.json`

```json
{
  "version": 1,
  "defaultProject": "76_in_КОРП",
  "projects": [
    { "id": 1, "name": "76", "fullName": "1. 76", "path": "D:\\1c\\EDT КОРП УОТ\\76", "roadmapDir": "1-76", ... },
    { "id": 2, "name": "76_in_КОРП", "fullName": "2. 76_in_КОРП", ... },
    { "id": 3, "name": "КОРП", "fullName": "3. КОРП", ... }
  ]
}
```

### 1.2. Единый источник задач: `tasks.json`

Каждый проект имеет `tasks.json` в своей папке `ДорожнаяКарта/N-проект/`. Структура:

| Поле | Описание |
|------|----------|
| `id` | Номер задачи |
| `name` | Название |
| `priority` | P0–P3 |
| `complexity` | 1–5 (⭐) |
| `progress` | 0–100 |
| `section` | `active` / `completed` / `postponed` |
| `stages[]` | Массив этапов с `id`, `name`, `desc`, `status`, `date` |

### 1.3. Представления: `.md` файлы

Генерируются из `tasks.json`. Редактирование только через `tasks.json`.

## 3. Команды управления

| Команда | Действие |
|---------|----------|
| `др` / `др1` / `сводка` / `дорожная карта` | Сводка всех проектов (активные + ✅ + ⏸️) |
| `др2` / `дор карта 2` | Детализация текущего проекта |
| `проект N` / `прN` | Переключение проекта |
| `стз N` / `статус задачи N` | Детализация задачи |
| `обнови статус N.M` | Изменить статус этапа |
| `закрываем задачу N` | ✅ Завершить задачу |
| `откладываем задачу N` | ⏸️ Отложить задачу |
| `итого время` | Сводка времени |
| `затрачено N мин на задачу M` | Вписать время |
| `отк отчет N.N` | Открыть HTML-отчёт |
| `синхронизация` | Синхронизировать проекты |

## 4. Правило отображения

При команде `др` — всегда показывать **все 3 раздела** для каждого проекта (активные, выполненные, отложенные). Полный список задач без сокращений.
