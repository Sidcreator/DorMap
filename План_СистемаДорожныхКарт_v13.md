## 1. Архитектура

Система дорожных карт состоит из следующих компонентов:

### 1.1. Компоненты

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **Единый список проектов** | `projects.json` | Список всех проектов, единый источник истины |
| **Задачи проекта** | `N-проект/tasks.json` | Задачи проекта, единый источник истины |
| **Дорожная карта (сводка)** | `ДорожнаяКарта.md` | Единая сводка по всем проектам |
| **Дорожная карта (проект)** | `N-проект/ДорожнаяКарта.md` | Детализация задач проекта |
| **Правила дорожной карты** | `ПравилаДорожнойКарты.md` | Полное руководство по работе с дорожной картой |
| **Правила AI** | `.roo/rules` | Инструкции для AI-ассистента (ключевые фразы) |
| **Журнал изменений** | `ЖурналИзменений.md` | История всех изменений с детализацией |

### 1.2. Единый источник проектов: `projects.json`

Список всех проектов в корне `ДорожнаяКарта/`. Формат:

```json
{
  "version": 1,
  "defaultProject": "1-76",
  "projects": [
    {
      "id": 1,
      "name": "76",
      "fullName": "1. Проект 76",
      "path": "D:\\1c\\EDT КОРП УОТ\\76",
      "roadmapDir": "1-76",
      "commands": ["проект 76", "проект 1", "пр76", "пр1"]
    }
  ]
}
```

Поля:
- `id` — номер проекта (1, 2, 3...)
- `name` — короткое имя для команд
- `fullName` — полное имя с номером для отображения
- `path` — полный путь к папке проекта в файловой системе
- `roadmapDir` — имя подпапки в `ДорожнаяКарта/`
- `commands` — список команд для переключения проекта

**Правило:** все изменения списка проектов — только в `projects.json`.

### 1.3. Единый источник задач: `tasks.json`

Каждый проект имеет `tasks.json` в своей папке `ДорожнаяКарта/N-проект/`. Структура задачи:

```json
{
  "id": 1,
  "name": "Название задачи",
  "priority": "P2",
  "complexity": 2,
  "progress": 0,
  "section": "active",
  "start": "2026-06-25",
  "end": null,
  "time_min": null,
  "stages": [
    { "id": "1.1", "name": "Этап", "desc": "Описание", "status": "⬜", "date": null, "report": null }
  ]
}
```

Поля секции:
- `section` — `"active"` / `"completed"` / `"postponed"` (раздел в дорожной карте)
- `stages[].status` — `⬜` (не начат), `🔵` (в работе), `✅` (выполнен), `🧪` (тестирование), `⚠️` (проблема), `🔒` (заблокирован), `⏭️` (пропущен)
- `stages[].report` — имя HTML-файла отчёта (опционально)
- Для `completed` задач: `start`, `end`, `time_min` (все заполнены)
- Для `postponed` задач: поле `reason` с причиной откладывания

**Workflow:**
1. Все изменения задач вносятся только в `tasks.json`
2. После изменения AI перегенерирует `ДорожнаяКарта.md` проекта
3. Команда `др` строит сводку из `tasks.json` (не из `.md`)

### 1.4. Структура папок

```
ДорожнаяКарта/                          ← корень системы
├── projects.json                       ← единый список проектов (источник истины)
├── ДорожнаяКарта.md                    ← единая сводка (выбор проекта + таблица задач)
├── ПравилаДорожнойКарты.md             ← правила ведения
├── README.md                           ← быстрый старт
├── План_СистемаДорожныхКарт_v*.md      ← версионные планы (полное описание, самодостаточные)
├── Новое в ДорожнойКарте_v*.md         ← описание изменений новой версии
├── Инструкция_Развертывание_без_AI_v*.md ← ручное развёртывание
├── 1-76/                               ← проект 1
│   ├── ДорожнаяКарта.md                ← детализация задач (представление из tasks.json)
│   ├── tasks.json                      ← задачи (единый источник истины)
│   ├── планы/                          ← планы разработки (опционально)
│   └── отчеты/                         ← HTML-отчёты, привязанные к этапам
├── 2-76_in_КОРП/                       ← проект 2
│   ├── ДорожнаяКарта.md
│   ├── tasks.json
│   ├── планы/
│   └── отчеты/
├── 3-КОРП/                             ← проект 3
│   ├── ДорожнаяКарта.md
│   ├── tasks.json
│   ├── планы/
│   └── отчеты/
```

## 2. Структура нумерации

| Уровень | Формат | Пример | Описание |
|---------|--------|--------|----------|
| 1 (задача) | `N` | `1` | Крупная задача |
| 2 (этап) | `N.N` | `1.1` | Стадия выполнения |
| 3 (шаг) | `N.N.N` | `1.2.1` | Конкретное действие |

Правила нумерации:
- Номера задач — последовательно, следующий свободный
- Активные задачи: 1, 2, 3... (максимум 5)
- Выполненные: группируются под одной задачей (напр. 7)
- Отложенные: группируются под одной задачей (напр. 8)
- При переносе из выполненных/отложенных в активные — новый активный номер

## 4. Команды управления

> Все команды вводятся в чат AI-ассистента. AI автоматически определяет команду по ключевой фразе и выполняет действия с дорожной картой.

### 4.1. Просмотр

| Команда | Действие |
|---------|----------|
| `др` / `др1` / `сводка` / `дорожная карта` / `итого` / `полная др` / `подробная` | Сводка **всех проектов** из `projects.json` с разделами: активные + ✅ + ⏸️ + итого |
| `др2` / `дор карта 2` | Детализация текущего проекта (уровни 2-3) |
| `только активные` | Только активные задачи (без ✅ и ⏸️) |
| `проект N` / `прN` / `пр N` / `проект Имя` | Переключиться на проект N |
| `стN` / `статус N` | Статус проекта N (без переключения) |
| `стз N` / `статус задачи N` | Детализация задачи N в текущем проекте |

### 4.2. Управление задачами

| Команда | Действие |
|---------|----------|
| `заводим задачу в дорожную карту` | Создать задачу. Проверить лимит 5 активных. Авто-отложить при превышении |
| `обнови статус N.M` | Изменить статус этапа, пересчитать % |
| `обнови статус N.M на ✅` | Сразу установить статус |
| `закрываем задачу N` | ✅ Пометить готово, запросить время (мин) |
| `откладываем задачу N` | ⏸️ Перенести в отложенные |
| `откладываем задачу N, причина: ...` | Отложить с указанием причины |
| `верни задачу N в активные` / `перенеси задачу N в активные` | Вернуть из выполненных/отложенных в активные с новым номером |

### 4.3. Учёт времени

| Команда | Действие |
|---------|----------|
| `затрачено N мин на задачу M` | Вписать время N в колонку задачи M |
| `итого время` | Сводка времени (мин / ч:мин) |

### 4.4. Отчёты

| Команда | Действие |
|---------|----------|
| `отк отчет` / `отк от` / `отк отчет текущий` / `отк от тек` | Открыть последний HTML-отчёт в браузере |
| `отк отчет ИмяФайла` | Открыть отчёт по имени из папки `отчеты/` |
| `отк отчет N.N` | Открыть отчёт, привязанный к этапу N.N (по колонке Отчёт в таблице этапов) |

### 4.5. Управление правилами

| Команда | Действие |
|---------|----------|
| `правила дорожной карты` | Показать `ПравилаДорожнойКарты.md` |
| `обновить правила дорожной карты` | Обновить все файлы системы, создать новую версию плана |
| `синхронизация` | Синхронизировать правила между всеми проектами |

## 5. Конвенции

### 5.1. Лимит активных задач
- Максимум **5 активных задач** (не отложенных) в дорожной карте
- При создании 6-й задачи без явного указания приоритета → автоматически отложить с причиной «Превышен лимит активных задач (максимум 5)»
- Если одна из 5 задач закрыта (✅) → можно вытащить задачу из отложенных

### 5.2. Учёт времени в минутах
- Единица измерения: минуты (целое число)
- Колонка «Время (мин)» — фактическое время, затраченное на задачу/этап
- Правила заполнения:
  - При старте задачи/этапа: `—` (прочерк)
  - При завершении: вписать количество минут (например: `15`, `120`, `5`)
  - При промежуточном статусе: `~N мин` (приблизительная оценка)
- Итог по времени: `N мин` или `N ч M мин` при N ≥ 60
- При закрытии задачи AI автоматически запрашивает затраченное время

### 5.3. Именование файлов
- **Вспомогательные файлы** (планы, инструкции, changelog) — с суффиксом `_vN`: `План_СистемаДорожныхКарт_v13.md`, `Инструкция_Развертывание_без_AI_v13.md`, `Новое в ДорожнойКарте_v13.md`
- **Основные рабочие файлы** — без суффикса: `ДорожнаяКарта.md`, `ПравилаДорожнойКарты.md`, `README.md`, `projects.json`, `tasks.json`
- Имена папок проектов: `N-КороткоеИмя` (например, `1-76`, `2-76_in_КОРП`)

### 5.4. `tasks.json` — единый источник истины
- Все изменения задач вносятся только в `tasks.json`
- `.md` файлы — представления, генерируются из `tasks.json`
- Команда `др` строит сводку из `tasks.json` в реальном времени

### 5.5. `projects.json` — единый источник проектов
- Список проектов — только в `projects.json`
- Все остальные файлы системы ссылаются на него

### 5.6. План версии — полное описание
- `План_СистемаДорожныхКарт_vN.md` — самодостаточный документ для развёртывания с нуля
- Должен содержать **полную спецификацию**: архитектуру, все правила, команды, конвенции, форматы, лимиты. Не только изменения.
- AI, получив этот файл, должен иметь достаточно информации, чтобы создать все файлы системы без обращения к другим источникам

### 5.7. Колонка «Отчёт» для этапов
- Каждый этап (уровень 2) может содержать ссылку на HTML-отчёт в колонке **Отчёт**
- Если отчёта нет — прочерк `—`
- Формат ссылки: `[Отчет_N.N_...html](отчеты/Отчет_N.N_...html)` — полное имя файла как текст ссылки
- Отчёты хранятся в папке `отчеты/` внутри проекта

## 7. Формат HTML-отчёта

### Шапка `<div class="meta">`
- Метод / endpoint
- Модуль / файл (если применимо)
- Строка в коде (если применимо)
- **Задача ДК** — номер задачи и этапа
- **Проект** — номер и имя проекта
- Дата создания

### Подвал
`Проект N/Имя — Дорожная карта. Отчёт привязан к задаче N.N.`

### CSS-классы для блоков
- `.issue` (красный) — проблемы
- `.ok` (зелёный) — успешные проверки
- `.warn` (оранжевый) — предупреждения

### Имя файла
`Отчет_N.N_КраткоеОписание.html`

Пример: `Отчет_4.1_Анализ_v1_analytics_stocks.html`

**Правило единообразия:** для всех проектов — полный список задач в каждом разделе, без сокращений.

## 10. Версионирование планов

### Правила
1. Файл плана всегда содержит версию в названии: `План_СистемаДорожныхКарт_vN.md` (N = номер версии)
2. Актуальная версия — файл с наибольшим номером N
3. При значительных изменениях:
   - Создать новый файл `План_СистемаДорожныхКарт_v(N+1).md` — **полное описание**, а не только изменения
   - Создать `Новое в ДорожнойКарте_v(N+1).md` — краткое описание изменений
   - Предыдущий файл остаётся без изменений (естественный архив)
4. Журнал версий — таблица в начале файла с колонками: Версия, Дата, Сессия, Изменения
5. Связь план↔карта: при каждом изменении плана проверять соответствие с `ДорожнаяКарта.md`

### Когда создавать новую версию
| Событие | Действие |
|---------|----------|
| Добавлен новый статус/символ | Новый файл v(N+1) |
| Изменена структура таблиц | Новый файл v(N+1) |
| Добавлена новая ключевая фраза | Новый файл v(N+1) |
| Изменён формат учёта времени | Новый файл v(N+1) |
| Изменена конвенция именования файлов | Новый файл v(N+1) |
| Добавлена новая категория команд | Новый файл v(N+1) |
| Добавлена новая колонка в таблицы | Новый файл v(N+1) |
| Добавлены новые папки/структура | Новый файл v(N+1) |
| Исправление опечатки | Без нового файла |
| Обновление примеров | Без нового файла |

### Правило синхронизации проектов
При любом изменении правил дорожной карты изменения применяются ко **всем проектам**:
- `1-76/ДорожнаяКарта.md`
- `2-76_in_КОРП/ДорожнаяКарта.md`
- `3-КОРП/ДорожнаяКарта.md`
- Единая `ДорожнаяКарта.md`

Процесс синхронизации:
1. Внести изменения в единые `ПравилаДорожнойКарты.md` и `План_СистемаДорожныхКарт_vN.md`
2. Обновить протокол управления (команды) в каждом проекте
3. Обновить структуру таблиц в каждом проекте
4. Обновить `ЖурналИзменений.md` каждого затронутого проекта
5. Обновить единую сводку

## 12. Шаблон файла .roo/rules

```markdown
## Roadmap & Task Management
- Roadmap file: ДорожнаяКарта/N-проект/ДорожнаяКарта.md
- Rules file: ДорожнаяКарта/ПравилаДорожнойКарты.md
- Change log: ДорожнаяКарта/N-проект/ЖурналИзменений.md
- Projects file: ДорожнаяКарта/projects.json
- Tasks file: ДорожнаяКарта/N-проект/tasks.json
- Numbering: N (task) → N.N (stage) → N.N.N (step)
- Key phrase «заводим задачу в дорожную карту» → Create new task in tasks.json → update .md
- Key phrase «др» / «др1» / «сводка» / «итого» → Show ALL projects from projects.json with active + ✅ + ⏸️ sections
- Key phrase «проект N» / «прN» → Switch to project N
- Key phrase «стз N» → Show task N details
- Key phrase «обнови статус N.M» → Change stage status, recalculate %
- Key phrase «закрываем задачу N» → Mark as ✅ Done, request time spent
- Key phrase «откладываем задачу N» → Mark as ⏸️ Deferred with reason
- Key phrase «итого время» → Show time summary in minutes
- Key phrase «затрачено N мин на задачу M» → Write time to tasks.json
- Key phrase «отк отчет» → Open last HTML report in browser
- Key phrase «отк отчет ИмяФайла» → Open report by name from отчеты/ folder
- Key phrase «отк отчет N.N» → Open report linked to stage N.N
- Key phrase «правила дорожной карты» → Show ПравилаДорожнойКарты.md
- Key phrase «обновить правила дорожной карты» → Update plan, rules, CONTEXT.md, all projects
- Key phrase «синхронизация» → Sync rules between all projects
- Always update tasks.json when completing stages (not .md directly)
- Track time: minutes (integer)
- Max 5 active tasks
```

## 13. Развёртывание (инструкция для AI)

При получении команды «Разверни систему дорожных карт по плану» AI выполняет:

### 13.1. Создать структуру папок

```
ДорожнаяКарта/
├── N-проект1/
│   ├── отчеты/
│   └── планы/
├── N-проект2/
│   ├── отчеты/
│   └── планы/
...
```

Где `N-проект` — для каждого проекта из запроса пользователя. Имена папок по шаблону `N-КороткоеИмя`.

### 13.2. Создать `projects.json`

По шаблону из раздела 1.2. Запросить у пользователя список проектов (id, name, fullName, path). Если пользователь не указал — создать с одним тестовым проектом.

### 13.3. Создать `ПравилаДорожнойКарты.md`

Включить полное содержимое разделов 2-12 данного плана (структура нумерации, статусы, команды, конвенции, форматы, лимиты, версионирование).

### 13.4. Создать `README.md`

Содержит:
- Краткое описание системы
- Таблицу команд (раздел 4 данного плана)
- Структуру папок (раздел 1.4)
- Быстрый старт: команда `др` → просмотр → управление

### 13.5. Создать `ДорожнаяКарта.md` (единая сводка)

Содержит:
- Выбор проекта (таблица из `projects.json`)
- Сводную таблицу задач (раздел 6)
- Протокол отображения (раздел 8)
- Итого по проектам

### 13.6. Для каждого проекта создать:

1. **`tasks.json`** — пустой или с одним примером задачи (по шаблону из раздела 1.3)
2. **`ДорожнаяКарта.md`** — сводная таблица задач + детализация этапов + протокол управления (по шаблону из раздела 6)
3. **Пустую папку `отчеты/`** и **пустую `планы/`**

### 13.7. Создать `Инструкция_Развертывание_без_AI_vN.md`

Содержит пошаговое руководство для человека (копирование файлов, настройка `projects.json`, инициализация, проверка).

### 13.8. Настроить `.roo/rules`

Добавить секцию из раздела 12, скорректировав пути к файлам под конкретную рабочую область.

### 13.9. Финальная проверка

Выполнить команду `др` — AI должен показать сводную таблицу всех проектов с разделами: активные + выполненные + отложенные.
