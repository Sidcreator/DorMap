## Шаг 2. Создать файл дорожной карты

Создать файл `docs/ДорожнаяКарта.md` (или в корне проекта) со следующей структурой:

### Обязательные секции:

### Конвенция: Лимит активных задач

- **Максимум 5 активных задач** (не отложенных) в дорожной карте
- При создании 6-й задачи без явного указания приоритета → **автоматически отложить** (перенести в секцию 7)
- Причина автооткладывания: «Превышен лимит активных задач (максимум 5)»
- Если одна из 5 задач закрыта (✅) → можно вытащить задачу из отложенных

### Конвенция: Перенос задач в активные

- При переносе задачи из выполненных или отложенных в активные задаче присваивается **новый номер активной задачи**.
- Новый номер выбирается по порядку среди активных задач: если активных задач нет, номер будет `1`; если активные задачи есть, используется следующий свободный номер.
- Старый номер фиксируется в описании задачи и журнале изменений как исторический источник переноса.
- Этапы, связанные планы и команды доступа перенумеровываются под новый активный номер; старый номер можно оставить алиасом, если по нему уже были ссылки.

### Конвенция: Отложенные задачи

- В **уровне 1** (дор карта 1) отложенные задачи отображаются **одной строкой**: `| **7** | ⏸️ Отложенные задачи (N шт.) |`
- В **уровне 2** (дор карта 2 / «статус задачи 7») — список всех отложенных задач с нумерацией `7.1`, `7.2`, `7.3`...
- В **уровне 3** — этапы каждой отложенной задачи: `7.1.1`, `7.1.2`...
- При команде «статус задачи 7» AI выдаёт все отложенные задачи с причинами и приоритетами
- При добавлении новой отложенной задачи: увеличить счётчик в уровне 1, добавить строку 7.N в уровень 2

### Конвенция: Отображение дорожной карты

- При любом запросе дорожной карты AI сначала показывает **сводную таблицу задач уровня 1**.
- Если пользователь запросил детализацию (`дор карта 2`, `статус задачи N` или аналогичный запрос), детализация выводится **после** сводной таблицы уровня 1.
- Фразы `дорожная карта`, `покажи дорожную карту`, `дор карта 1`, `итого`, `сводка` считаются запросом сводной таблицы уровня 1.

### Конвенция: Учёт времени в минутах

- **Единица измерения:** минуты (целое число)
- **Колонка «Время (мин)»** — фактическое время, затраченное на задачу/этап
- **Правила заполнения:**
  - При старте задачи/этапа: `—` (прочерк)
  - При завершении: вписать количество минут (например: `15`, `120`, `5`)
  - При промежуточном статусе: `~N мин` (приблизительная оценка, если точное время неизвестно)
- **Итог по времени** — секция в конце дорожной карты:
  - Суммарное время по каждой задаче
  - Суммарное время за период (день/неделя)
  - Формат: `N мин` или `N ч M мин` при N ≥ 60
- **Команда «итого время»** — AI автоматически суммирует все значения из колонки «Время (мин)» и выводит сводку
- **Автоматический подсчёт:** при закрытии задачи AI предлагает вписать затраченное время или оценить его

```markdown
# Дорожная карта проекта [ИМЯ]

## Структура нумерации
| Уровень | Формат | Пример | Описание |
|---------|--------|--------|----------|
| 1 (задача) | N | 1 | Крупная задача |
| 2 (этап) | N.N | 1.1 | Стадия выполнения |

| 1.1 | [Этап] | [Описание] | ⬜ | | — | |

## 6. ✅ Выполненные задачи
> В уровне 1 — одна строка: `| **6** | ✅ Выполненные задачи (N шт.) | — | — | — | ✅ | — | — | — |`

| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
|---|--------|---|-----------|---|--------|--------|-------|-------------|

## 7. ⏸️ Отложенные задачи
> В уровне 1 — одна строка: `| **7** | ⏸️ Отложенные задачи (N шт.) | — | — | — | ⏸️ | — | — | — |`

|-------|----------|
| «заводим задачу в дорожную карту» | Создать задачу → дор карта 1 |
```

## Шаг 3. Настроить правила AI (.roo/rules)

Добавить в файл `.roo/rules` следующую секцию:

```markdown
## Roadmap & Task Management
- Roadmap file: [путь к ДорожнаяКарта.md]
- Rules file: [путь к ПравилаДорожнойКарты.md]
- Change log: [путь к ЖурналИзменений.md]
- Numbering: N (task) → N.N (stage) → N.N.N (step)
- Key phrase «заводим задачу в дорожную карту» → Create new task, add to дор карта 1
- Key phrase «дор карта 1» / «итого» / «сводка» / «дорожная карта» / «покажи дорожную карту» → Show summary table (level 1)
- Key phrase «дор карта 2» → Show summary table first, then all task details (levels 2-3)
- Key phrase «статус задачи N» → Show task N details
- Key phrase «обнови статус N.M» → Change stage status, recalculate %
- Key phrase «закрываем задачу N» → Mark as ✅ Done, request time spent
- Key phrase «откладываем задачу N» → Mark as ⏸️ Deferred
- Key phrase «верни задачу N в активные» / «перенеси задачу N в активные» → Move done/deferred task to active with a new active sequence number
- Key phrase «итого время» → Show time summary in minutes
- Key phrase «затрачено N мин на задачу M» → Write time to task M
- Key phrase «план разработки N» / «план разработки N.X» → Show linked development plan from roadmap
- Key phrase «правила дорожной карты» → Show or generate ПравилаДорожнойКарты.md
- Key phrase «обновить правила дорожной карты» → Update plan, rules, CONTEXT.md, ЖурналИзменений.md
- Always update roadmap when completing stages
- Always start any roadmap display response with the summary table (level 1)
- Always update ЖурналИзменений.md for code changes
- Track time: record start/end date, time in MINUTES for each task and stage
- Time unit: minutes (integer). Format: «N мин» or «N ч M мин» if ≥ 60
- Maintain ⏸️ Deferred tasks list with reason
```

## 📖 Руководство пользователя: Команды дорожной карты

> Все команды вводятся в чат AI-ассистента (Roo Code). AI автоматически определяет команду по ключевой фразе и выполняет действия с дорожной картой.
>
> **Важно:** каждая команда выполняет строго своё действие. Команда `«заводим задачу»` добавляет задачу в «дор карта 1» — она не показывает статус, не обновляет этапы и не закрывает задачи. Для каждого действия — своя команда.
>
> **Правило показа:** любой запрос дорожной карты начинается со сводной таблицы уровня 1. Детализация, если нужна, выводится после неё.

### Категории команд

| Категория | Команды | Назначение |
|-----------|---------|------------|
| 🆕 **Создание** | `заводим задачу` | Добавить новую задачу в «дор карта 1» |
| 👁️ **Просмотр** | `дор карта 1`, `дор карта 2`, `дорожная карта`, `статус задачи N`, `план разработки N` | Посмотреть состояние карты и связанные планы |
| ✏️ **Изменение** | `обнови статус N.M` | Изменить статус этапа |
| ✅ **Завершение** | `закрываем задачу N`, `откладываем задачу N`, `верни задачу N в активные` | Закрыть, отложить или вернуть в активные |
| ⏱️ **Время** | `итого время`, `затрачено N мин` | Учёт затраченного времени |
| ⚙️ **Управление правилами** | `правила дорожной карты`, `обновить правила дорожной карты` | Просмотр и обновление правил системы |

### Подробное описание команд по категориям

---

### 🆕 Категория: Создание

#### `«заводим задачу в дорожную карту»`

**Синтаксис:**
```
заводим задачу в дорожную карту: [название], приоритет [P], этапы: [список]
```

**Что делает AI:**
1. Назначает следующий свободный номер задачи
2. Создаёт этапы с нумерацией N.1, N.2, N.3...
3. **Добавляет задачу в «дор карта 1»** (сводную таблицу уровня 1)
4. Добавляет в канбан-доску в секцию «📋 Планируется»
5. Проверяет лимит активных задач (максимум 5)
6. Если лимит превышен → автоматически откладывает с причиной

**Пример:**
```
заводим задачу в дорожную карту: Настроить CI/CD, приоритет P2, этапы:
1) Настроить GitLab Runner
2) Написать .gitlab-ci.yml
3) Протестировать пайплайн
```

**Результат:** В «дор карта 1» появится новая строка:
```
| 1 | Настроить CI/CD | P2 | ⭐⭐⭐ | 0% | 📋 | 2026-05-16 | — | — |
```

---

### 👁️ Категория: Просмотр

#### `«дор карта 1»`, `«итого»`, `«сводка»`, `«дорожная карта»` или `«покажи дорожную карту»`

**Что делает AI:** Показывает сводную таблицу задач уровня 1.

**Результат:** Таблица вида:
```
| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
```

---

#### `«дор карта 2»`

**Что делает AI:** Сначала показывает сводную таблицу уровня 1, затем полную детализацию всех задач (уровни 2 и 3) — все этапы и шаги.

---

#### `«статус задачи N»`

**Что делает AI:** Показывает детальную информацию по одной задаче N.

**Пример:** `статус задачи 6` — покажет все этапы задачи 6.
**Особенность:** `статус задачи 7` — покажет все отложенные задачи с причинами.

---

### `«план разработки N»` / `«план разработки N.X»`

**Что делает AI:** Сначала показывает сводную таблицу уровня 1, затем находит в дорожной карте связанный план разработки и показывает общий индекс или конкретный раздел.

**Пример:** `план разработки 6.8` показывает пакет планов внедрения процесса заказов Ozon; `план разработки 6.8.P7` показывает план отгрузки FBO.

---

### ✏️ Категория: Изменение

#### `«обнови статус N.M»`

**Синтаксис:**
```
обнови статус 2.1          → AI спросит новый статус
обнови статус 2.1 на ✅    → AI сразу установит статус
```

**Что делает AI:**
1. Находит этап N.M в дорожной карте
2. Меняет статус (⬜ → 🔵 → ✅ и т.д.)
3. Пересчитывает процент выполнения задачи N
4. Обновляет статус задачи в «дор карта 1»
5. Обновляет канбан-доску

---

### ✅ Категория: Завершение

#### `«закрываем задачу N»`

**Что делает AI:**
1. Устанавливает все оставшиеся этапы в ✅
2. Устанавливает % = 100%
3. Записывает дату завершения
4. **Запрашивает затраченное время** в минутах
5. Переносит задачу в секцию «6. ✅ Выполненные задачи»
6. Обновляет «дор карта 1» и канбан

**Пример диалога:**
```
Пользователь: закрываем задачу 3
AI: Задача 3 закрыта ✅. Сколько минут затрачено?
Пользователь: 45
AI: Время зафиксировано: 45 мин. Задача перенесена в секцию 6.
```

---

#### `«откладываем задачу N»`

**Синтаксис:**
```
откладываем задачу 3
откладываем задачу 3, причина: жду ответ от клиента
```

**Что делает AI:**
1. Переносит задачу в секцию «7. ⏸️ Отложенные задачи»
2. Увеличивает счётчик в «дор карта 1»
3. Записывает причину откладывания
4. Обновляет канбан

---

### `«верни задачу N в активные»` / `«перенеси задачу N в активные»`

**Что делает AI:** переносит задачу из выполненных или отложенных в активные и присваивает ей новый активный номер по порядку.

**Правило номера:** если активных задач нет, новый номер будет `1`; если активные задачи есть, используется следующий свободный номер.

**Дополнительно:** старый номер фиксируется как исторический источник, связанные этапы/планы перенумеровываются под новый номер.

### ⏱️ Категория: Время

#### `«итого время»`

**Что делает AI:**
1. Суммирует все значения из колонки «Время (мин)»
2. Группирует по периодам (дням)
3. Конвертирует в удобный формат (мин → ч:мин при ≥ 60)

**Пример результата:**
```
| Период | Задач завершено | Время (мин) | Время (ч:мин) |
|--------|----------------|-------------|----------------|
| 2026-05-16 | 8 | 245 | 4 ч 5 мин |
```

---

#### `«затрачено N мин на задачу M»`

**Что делает AI:**
1. Находит задачу M в дорожной карте
2. Вписывает значение N в колонку «Время (мин)»
3. Обновляет «Итог по времени»

**Пример:** `затрачено 30 мин на задачу 2`

## Версионирование планов

### Правила

1. **Файл плана всегда содержит версию в названии:** `План_СистемаДорожныхКарт_vN.md` (N = номер версии)
2. **Актуальная версия** — файл с наибольшим номером N (например, `План_СистемаДорожныхКарт_v8.md`)
3. При значительных изменениях плана:
   - Создать новый файл `План_СистемаДорожныхКарт_v(N+1).md` на основе текущего
   - В новом файле обновить заголовок, версию, журнал версий
   - В новом файле добавить ссылку на предыдущую версию
   - Предыдущий файл остаётся без изменений (естественный архив)
4. **Журнал версий** — таблица в начале файла с колонками: Версия, Дата, Сессия, Изменения
5. **Связь план↔карта**: при каждом изменении плана проверять соответствие с `ДорожнаяКарта.md`

### Когда создавать новую версию

| Событие | Действие |
|---------|----------|
| Добавлен новый статус/символ | Новый файл v(N+1) |
| Изменена структура таблиц | Новый файл v(N+1) |
| Добавлена новая ключевая фраза | Новый файл v(N+1) |
| Изменён формат учёта времени | Новый файл v(N+1) |
| Изменена конвенция именования файлов | Новый файл v(N+1) |
| Добавлена новая категория команд | Новый файл v(N+1) |
| Изменено правило отображения дорожной карты | Новый файл v(N+1) |
| Изменено правило переноса задач между разделами | Новый файл v(N+1) |
| Исправление опечатки | Без нового файла |
| Обновление примеров | Без нового файла |

### Соответствие план ↔ дорожная карта

При рассинхронизации (дорожная карта эволюционировала быстрее плана):
1. Зафиксировать расхождение в журнале версий
2. Создать новый файл плана v(N+1) с актуальными данными
3. Проверить соответствие всех ключевых фраз
