2.5 Асинхронный отчёт (create → poll → download)

НЕТ РЕАЛИЗОВАННОГО ПРИМЕРА В УОТ614. Паттерн требуется разработать с нуля:
  1. POST /v1/report/discounted/create → получаем code (UUID отчёта)
  2. GET /v1/report/info?code={uuid} → опрашиваем статус (IN_PROGRESS → ждём → OK/ERROR)
  3. GET /v1/report/info/download?code={uuid} → скачиваем CSV-файл

7. Риски

1. Асинхронный отчёт (4.2): Паттерн create → poll → download не имеет готового аналога в УОТ614. Механизм опроса через Пауза() может блокировать длительные операции. Рекомендуется через CG_Online_ДлительныеОперацииПереопределяемый или внешний цикл с проверкой статуса.
2. Динамический ответ /v1/analytics/data: Состав полей меняется в зависимости от запроса. Нужна динамическая таблица значений или универсальный маппинг.
3. Rate limit 1 запрос/мин: Для /v1/analytics/data нужно хранить timestamp последнего запроса и блокировать повторные.
4. /v2/analytics/stock_on_warehouses — deprecated: Не требует реализации. Используем существующий /v1/analytics/stocks.
