Метод: POST /v1/analytics/data
Наличие в УОТ614: НЕТ
Задача ДК: 4.7 — /v1/analytics/data - Данные аналитики
Проект: 1/76
Дата: 2026-06-25

1. Описание

Метод возвращает аналитику по товарам продавца, сгруппированную по указанному измерению (dimension). Соответствует разделу Аналитика → Графики в личном кабинете Ozon.

Ограничения:
• Без подписки Premium Plus — данные только за последние 3 месяца, ограниченные метрики и способы группировки
• Rate limit: 1 запрос в минуту
• Максимум 14 метрик в одном запросе
• limit: max 1000, min 1

6. Пример запроса

{ "date_from": "2020-09-01", "date_to": "2021-10-15", "metrics": [ "hits_view_search" ], "dimension": [ "sku", "day" ], "filters": [], "sort": [ { "key": "hits_view_search", "order": "DESC" } ], "limit": 1000, "offset": 0 }

7. Схема ответа

{ "result": { "data": [], "totals": [ 0 ] }, "timestamp": "2021-11-25 15:19:21" }
ПолеТипОписание
resultobjectКонтейнер с данными
result.dataarrayМассив данных (пустой в примере — структура элемента пока не определена)
result.totalsarray[int64]Итоговые значения по метрикам (порядок соответствует metrics[] в запросе)
timestampstringВремя формирования ответа (формат: "YYYY-MM-DD HH:mm:ss")
Проблема: result.data в примере пустой. Требуется ответ с реальными данными для определения структуры элемента (ожидается: dimension-поля + метрики для каждой группы).
