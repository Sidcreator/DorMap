2. Выявленные проблемы

Проблема 1: Поле reserved_amount отсутствует в JSON ответа
Строка 9324: НоваяСтрока.КоличествоВРезерве = СтрокаОстатка["reserved_amount"]
В ответе API нет поля reserved_amount. Всё, что связано с резервами: stock_defect_stock_count (брак), excess_stock_count (избыток).
Результат: КоличествоВРезерве всегда Неопределено/0.
Проблема 2: requested_stock_count закомментирован
Строка 9321 закомментирована. Вместо него на КоличествоВБудущихПоставках маппится transit_stock_count (товары в пути), хотя requested_stock_count — заявки на поставку.
Проблема 3: Отсутствуют поля фильтрации в запросе
Из 6 полей запроса передаётся только skus. Нет пагинации (limit/offset).
