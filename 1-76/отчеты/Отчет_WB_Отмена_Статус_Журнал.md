# Отчет: WB FBS, отмена заказа и отображение статуса в журнале

> **Дата анализа:** 2026-05-20  
> **Задача:** выяснить, почему при тесте загрузки заказа Wildberries с отменой не заполняется колонка `СписокДокументыСервисовСтатус` в форме `CG_Online_IWS_ЖурналДокументыСервисов.СписокДокументов`.  
> **Статус задачи:** анализ выполнен, задача не закрыта; требуется корректировка тестовых макетов и повторная проверка.

---

## Краткий вывод

Причина не в форме журнала. Поле `СписокДокументыСервисовСтатус` не заполняется, потому что в новом тестовом сценарии отмены данные макета заказов и макета статусов WB не синхронизированы по `id` заказа.

Для корректного отображения отмены заказ должен одновременно присутствовать:

1. в ответе `api/v3/orders`;
2. в ответе `api/v3/orders/status`;
3. с парой `supplierStatus + wbStatus`, которая есть во внутреннем макете статусов УОТ607 и сопоставляется со статусом `Отменено = Истина`.

Для текущего сценария наиболее прямой вариант: вернуть заказ `3059523133` в макеты `CG_Online_Wildberries_JSON_ЗагрузитьЗаказыССобственногоСкладаСОтменой_Краткий` и, при необходимости, `_Полный`.

## Как работает цепочка статуса

1. Форма журнала вызывает загрузку заказов с собственного склада WB.
2. Обработка WB получает список заказов через `api/v3/orders` и формирует `тзЗаказы`.
3. Затем вызывается получение статусов через `api/v3/orders/status`.
4. Для каждого статуса выполняется поиск строки заказа:

```bsl
НайденныеСтроки = тзЗаказы.НайтиСтроки(Отбор);
```

5. Если строка найдена, в неё записываются:

```bsl
НайденнаяСтрока.Статус = Статус;
НайденнаяСтрока.Отменено = Статус.Отменено;
```

6. Далее создаётся или обновляется документ сервиса, его статус попадает в регистр `CG_Online_РеестрДокументовСервисов`.
7. Форма `СписокДокументов` показывает `СписокДокументыСервисов.Статус`, который берётся из регистра реестра.

Если на шаге 4 заказ не найден в `тзЗаказы`, статус из ответа WB не применяется и в журнале отображаться нечему.

## Сочетания статусов WB для отмены

Внутренний макет статусов WB содержит отменные сочетания:

| `supplierStatus` | `wbStatus` | Результат |
|---|---|---|
| `cancel` | `canceled` | отменено поставщиком, `Отменено = Истина` |
| `cancel_missed_call` | `canceled_by_missed_call` | отмена по недозвону, `Отменено = Истина` |
| `cancel_missed_call` | пусто | отмена по недозвону, `Отменено = Истина` |
| `new` | `declined_by_client` | покупатель отменил в первый час, `Отменено = Истина` |
| `confirm` | `declined_by_client` | покупатель отменил в первый час, `Отменено = Истина` |
| `complete` | `defect` | отменено по браку, `Отменено = Истина` |

Для заказа `3059523133` в кратком макете статусов с отменой используется корректная пара:

```text
supplierStatus = cancel
wbStatus       = canceled
```

Эта пара должна сопоставиться с внутренним статусом отмены, если заказ `3059523133` есть в `тзЗаказы`.

Для полного макета требуется отдельная проверка: если у `3059523133` там осталось сочетание `complete + canceled`, оно не найдено во внутреннем макете статусов WB. В этом случае одного возврата заказа в макет заказов будет недостаточно.

## Почему форма журнала не является причиной

Элемент формы `СписокДокументыСервисовСтатус` привязан к данным динамического списка:

```text
СписокДокументыСервисов.Статус
```

Запрос динамического списка при инициализации формы подменяется общим запросом `ТекстЗапросаРеестрДокументовСервисов()`, где статус берётся из регистра:

```text
РегистрСведений.CG_Online_РеестрДокументовСервисов.Статус
```

То есть форма отображает то, что было записано в документ и реестр. В текущем тесте статус не доходит до документа, потому что отменный статус не применяется к строке `тзЗаказы`.

## Рекомендация для продолжения

Для проверки корректного отображения отмены в журнале:

1. Вернуть заказ `3059523133` в макет `CG_Online_Wildberries_JSON_ЗагрузитьЗаказыССобственногоСкладаСОтменой_Краткий`.
2. Для полного сценария также вернуть его в `..._Полный` либо выбрать другой отменный `id`, который есть в полном макете заказов.
3. Убедиться, что в макете статусов для выбранного `id` используется пара, существующая во внутреннем макете статусов WB. Для `3059523133` предпочтительно использовать `cancel + canceled`.
4. Повторить тест загрузки заказа с отменой.
5. В отладке проверить точку:

```bsl
НайденныеСтроки = тзЗаказы.НайтиСтроки(Отбор);
```

Ожидаемый результат после синхронизации макетов: для `id = 3059523133` должна находиться строка заказа, после чего статус записывается в `тзЗаказы`, документ сервиса и регистр реестра.
