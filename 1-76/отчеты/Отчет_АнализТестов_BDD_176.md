## 5. Качественный анализ

### 5.1. Сильные стороны ✅

1. **Полнота покрытия основных бизнес-процессов:** загрузка товаров, заказов, возвратов, выгрузка товаров/остатков
2. **Параметризованные экспортные сценарии** (серия 050xx) — переиспользуемые компоненты
3. **Глобальные переменные** для сервисов, схем, режимов — легко менять параметры
4. **Четыре маркетплейса** в покрытии: Ozon, WB, YM, SMM
5. **Три схемы работы** для заказов: FBS, FBO, FBM
6. **Детальный тест выгрузки товаров** (01140, 1231 строка) — самый проработанный
7. **Проверка маркировки** при генерации макета и сохранении заказа
8. **Проверка доступности меню** (06000) — дымовой тест интерфейса

### 5.2. Проблемы и риски ⚠️

1. **Отсутствие проверок результата в большинстве загрузочных тестов (01020–01090)**
   - Все 8 файлов используют один и тот же шаблон: открыть журнал → выбрать Ozon → нажать кнопку
   - Проверки `ЕстьСообщенияПользователюТоОшибка` закомментированы
   - Тесты «зелёные» даже если загрузка не создала ни одного документа

2. **СберМегаМаркет — минимальное покрытие**
   - В 01100 ID складов SMM закомментирован
   - В 02010 только FBS, FBO/FBM закомментированы
   - В 05160 проверки результата закомментированы

3. **Нет тестов для Avito** — критический пробел, т.к. ИС_AVITO — одно из расширений проекта

4. **Жёсткая привязка к данным**
   - Захардкожены: «Ozon 1 Рабочий», «Тестовая контора», «Партнер Озон ООО 1»
   - Захардкожены категории: «Детские аксессуары», «Украшения для девочек»
   - Захардкожены штрихкоды: «4610169736014»

5. **Нет очистки тестовых данных** — тесты накапливают данные, повторный запуск может давать другой результат

6. **Нет проверки данных документов** — загруженные заказы не проверяются на корректность реквизитов

7. **Много закомментированного кода** — снижает читаемость и поддерживаемость

## 8. Вывод

Пул тестов представляет собой **функциональный набор для регрессионного тестирования** УОТ604 с покрытием 4 маркетплейсов. Основной упор сделан на **загрузку заказов** (самая проработанная часть — 4 параметризованных Export-файла по ~450 строк каждый) и **выгрузку товаров** (детальный тест на 1231 строку).

**Ключевой пробел — отсутствие тестов для Avito**, что критично в контексте проекта по интеграции ИС_AVITO в УОТ604. Также значительная часть загрузочных тестов (01020–01090) фактически является «нажимательными» тестами без валидации результата.
