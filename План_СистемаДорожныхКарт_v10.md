## Обзор системы

Система состоит из 5 компонентов:

| Компонент | Файл | Назначение |
|-----------|------|------------|
| **Единая сводка** | `ДорожнаяКарта.md` | Выбор проекта + сводная таблица всех проектов |
| **Дорожная карта проекта** | `N-проект/ДорожнаяКарта.md` | Детализация задач конкретного проекта |
| **Правила дорожной карты** | `ПравилаДорожнойКарты.md` | Полное руководство по работе с дорожной картой |
| **Правила AI** | `.roo/rules` | Инструкции для AI-ассистента (ключевые фразы) |
| **Журнал изменений** | `N-проект/ЖурналИзменений.md` | История всех изменений с детализацией |

> **Важно:** Старые отдельные файлы дорожных карт (например, `1сexpert/docs/ДорожнаяКарта.md`) **не используются**. Источник истины — только единая `ДорожнаяКарта/ДорожнаяКарта.md` и подпапка `N-проект/`.

## Шаг 1. Создать структуру каталогов (единая папка ДорожнаяКарта/)

Все файлы дорожной карты ведутся в единой папке `ДорожнаяКарта/`. Каждый проект получает подпапку с номером.

```
ДорожнаяКарта/                          — корень (общий для всех проектов)
├── ДорожнаяКарта.md                    — единая сводка: выбор проекта + таблица уровня 1
├── ПравилаДорожнойКарты.md             — полное руководство
├── План_СистемаДорожныхКарт_vN.md     — версионированный план
├── 1-имя_проекта/                      — проект 1
│   ├── ДорожнаяКарта.md                — детализация задач проекта
│   ├── ЖурналИзменений.md              — история изменений
│   ├── tasks.json                      — машиночитаемые данные
│   ├── plans/                          — планы разработки
│   └── docs/                           — HTML-отчёты и документы
├── 2-имя_проекта/                      — проект 2
└── 3-имя_проекта/                      — проект 3
└── (следующий номер при добавлении)
```

### Архивация перед переносом

Чтобы избежать потери данных при переносе файлов старой дорожной карты в новую структуру — **обязательно сформировать ZIP-архив** всех файлов дорожной карты **до начала переноса**. Архив расположить в **старой директории** проекта.

Порядок:
1. Создать ZIP-архив всех файлов дорожной карты в старой директории (например, `1сexpert/ДорожнаяКарта_архив_YYYY-MM-DD.zip`)
2. После создания архива — выполнять перенос файлов в новую структуру `ДорожнаяКарта/`
3. В журнале изменений указать ссылку на архив

### Формат колонки «Проект»

Колонка «Проект» во всех таблицах содержит:
- Название проекта в формате `N. Имя`
- Путь к проекту на следующей строке: `D:\путь\к\проекту\`

Шаблон: **N. Имя**<br>`D:\путь\к\проекту\`

Привязка формируется в момент добавления нового проекта или изменения директории проекта.

## Шаг 2. Создать единую сводку ДорожнаяКарта.md

### Секция «Выбор проекта»

```markdown
| # | Проект | Дорожная карта | Команда |
|---|--------|----------------|---------|
| **1** | **1. Имя**<br>`D:\путь\` | [`1-имя/ДорожнаяКарта.md`](1-имя/ДорожнаяКарта.md) | `проект 1` / `проект 1/имя` |
```

### Сводная таблица задач (Уровень 1)

Колонка «Проект» — первая слева. Проекты отсортированы по номеру.

```markdown
| Проект | № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
|--------|---|--------|---|-----------|---|--------|--------|-------|-------------|
| **1. Имя**<br>`D:\путь\` | **1** | Название задачи | P1 | ⭐⭐⭐ | 0% | 🔧 | 2026-06-01 | — | — |
```

### Итого по проектам

```markdown
| Проект | Активных | Готово | Отложено | Прогресс |
|--------|---------|--------|----------|----------|
| **1. Имя**<br>`D:\путь\` | N | N | N | — |
```

## Шаг 3. Файл дорожной карты проекта (N-проект/ДорожнаяКарта.md)

Структура та же, что в v8, но с ключевыми изменениями:

### Нумерация проектов

- Каждый проект имеет номер: `1. 76`, `2. 76_in_КОРП`, `3. КОРП`
- При добавлении нового проекта — номера пересчитываются по порядку
- При удалении проекта — номера оставшихся сохраняются (зазоров не возникает)
- Папки проектов именуются `N-имя_проекта`

### Протокол добавления нового проекта

1. Запросить у пользователя путь к проекту (полный путь до папки)
2. **Спросить, нужно ли перенести файлы дорожной карты из старого проекта** (возможно, это не новый проект, а перенос существующего). Если да — запросить путь к старой папке с файлами дорожной карты.
3. Зафиксировать путь в колонке «Проект» (формат: `N. Имя`<br>`путь`)
4. Создать подпапку `ДорожнаяКарта/N-имя_проекта/`
5. Если есть старые файлы — перенести их в новую подпапку
6. Перенумеровать проекты по порядку
7. Обновить `ДорожнаяКарта.md`, `ПравилаДорожнойКарты.md`, `tasks.json`, `.roo/rules`, `CONTEXT.md`

## Шаг 4. Настроить правила AI (.roo/rules)

```markdown
## Roadmap & Task Management
- Short commands: `др1` (дор карта 1), `др2` (дор карта 2), `пр1`/`пр2`/`пр3` (проект 1/2/3), `ст1`/`ст2`/`ст3` (статус проекта 1/2/3), `стз N` (статус задачи N в текущем проекте)
- Roadmap location: ДорожнаяКарта/ДорожнаяКарта.md (unified for all projects)
- Current project: [текущий проект]. Subfolder: ДорожнаяКарта/N-имя_проекта/
- Change log: ДорожнаяКарта/N-имя_проекта/ЖурналИзменений.md
- Task data: ДорожнаяКарта/N-имя_проекта/tasks.json
- Key phrase «дор карта 1» / «итого» / «сводка» / «др1» → Show **full** level 1 table — все проекты, все задачи, все колонки
- Key phrase «дор карта 2» / «др2» → Show active tasks of the current project only
- Key phrase «прN» / «проект N» → Switch context to project N
- Key phrase «стN» / «статус N» → Show status of project N
- Key phrase «стз N» / «статус задачи N» → Show task N details in the **currently selected project** (does not change context)
```

## Шаг 5. Короткие команды

| Команда | Действие |
|---------|----------|
| `др1` | Полная сводная таблица уровня 1 (все проекты, все задачи, все колонки) |
| `др2` | Активные задачи текущего проекта |
| `пр1` / `пр2` / `пр3` | Переключение проекта 1/2/3 |
| `ст1` / `ст2` / `ст3` | Статус проекта 1/2/3 (просмотр, контекст не меняется) |
| `стз N` | Детализация задачи N в текущем выбранном проекте |

## Шаг 6. Создать первую задачу

Сказать AI-ассистенту:

> **«заводим задачу в дорожную карту: [название задачи], приоритет P1, проект 1, этапы: [список этапов]»**

AI автоматически:
1. Определяет текущий проект (или по явному указанию)
2. Назначает следующий свободный номер
3. Создаёт этапы с нумерацией N.1, N.2, ...
4. Добавляет задачу в сводную таблицу уровня 1 единой `ДорожнаяКарта.md`
5. Добавляет в детализацию проекта `N-проект/ДорожнаяКарта.md`
6. Добавляет в канбан «Планируется»

## Статусы и их обозначения

### Задачи (уровень 1)

| Символ | Статус | Когда ставить |
|--------|--------|---------------|
| 📋 | Планируется | Задача создана, ещё не начата |
| 🔍 | Анализ | Начат первый этап (анализ) |
| 🔧 | Разработка | Идёт написание кода |
| 🧪 | Тестирование | Код написан, ожидает проверки |
| 🔒 | Заблокировано | Ожидает внешнего фактора |
| ✅ | Готово | Все этапы завершены |
| ⏸️ | Отложено | Задача отложена |

### Этапы (уровни 2-3)

| Символ | Статус |
|--------|--------|
| ⬜ | Не начат |
| 🔵 | В работе |
| ✅ | Выполнен |
| 🧪 | Тестирование |
| ⚠️ | Проблема |
| 🔒 | Заблокирован |
| ⏭️ | Пропущен |

| Фраза | Действие |

## Версионирование планов

### Правила

1. **Файл плана всегда содержит версию в названии:** `План_СистемаДорожныхКарт_vN.md`
2. **Актуальная версия** — файл с наибольшим номером N
3. При значительных изменениях:
   - Создать новый файл `План_СистемаДорожныхКарт_v(N+1).md`
   - Обновить заголовок, версию, журнал версий
   - Добавить ссылку на предыдущую версию
4. **Журнал версий** — таблица в начале файла

### Когда создавать новую версию

| Событие | Действие |
|---------|----------|
| Добавлен проект | Новый файл v(N+1) |
| Удалён проект | Новый файл v(N+1) |
| Изменена структура папок | Новый файл v(N+1) |
| Добавлена новая команда | Новый файл v(N+1) |
| Изменён формат таблиц | Новый файл v(N+1) |
| Добавлена новая конвенция | Новый файл v(N+1) |
| Исправление опечатки | Без нового файла |
