# Правила пользования дорожной картой

> Полное руководство по работе с системой дорожных карт проектов.
> Версия: v13 (2026-06-28)

---

## 1. Общие положения

### Единый источник проектов

Список всех проектов — в `projects.json`. Поля:
- `id` — номер проекта
- `name` — короткое имя
- `fullName` — полное имя с номером
- `path` — полный путь к директории проекта
- `roadmapDir` — имя папки в `ДорожнаяКарта/`
- `roadmapFile` — путь к файлу дорожной карты
- `commands` — список команд для переключения

### Единый источник задач

Каждый проект имеет `tasks.json` в своей папке. Все изменения задач — только в `tasks.json`. `.md` файлы генерируются из него.

## 3. Статусы

### Задачи (уровень 1)

| Символ | Статус | Описание |
|--------|--------|----------|
| 📋 | Планируется | Задача создана, не начата |
| 🔍 | Анализ | Начат первый этап |
| 🔧 | Разработка | Идёт написание кода |
| 🧪 | Тестирование | Ожидает проверки |
| 🔒 | Заблокировано | Внешний фактор |
| ✅ | Готово | Все этапы завершены |
| ⏸️ | Отложено | Отложена по приоритету |

### Этапы (уровни 2-3)

| Символ | Статус |
|--------|--------|
| ⬜ | Не начат |
| 🔵 | В работе |
| ✅ | Выполнен |
| 🧪 | Тестирование |
| ⚠️ | Проблема |
| 🔒 | Заблокирован |
| ⏭️ | Пропущен |

### Приоритеты

| Приоритет | Описание |
|-----------|----------|
| P0 | Критический, блокирует работу |
| P1 | Высокий, в первую очередь |
| P2 | Средний, важно |
| P2.5 | Между P2 и P3 |
| P3 | Низкий, желательно |

## 4. Команды управления

| Категория | Команда | Действие |
|-----------|---------|----------|
| **Просмотр** | `др` / `др1` / `сводка` / `итого` | Сводка всех проектов (активные + ✅ + ⏸️) |
| | `др2` / `дор карта 2` | Детализация текущего проекта |
| | `проект N` / `прN` | Переключиться на проект N |
| | `стз N` / `статус задачи N` | Детализация задачи N |
| **Управление** | `заводим задачу` | Создать задачу (макс 5 активных) |
| | `обнови статус N.M` | Изменить статус этапа |
| | `закрываем задачу N` | ✅ Пометить готово, запросить время |
| | `откладываем задачу N` | ⏸️ Отложить с причиной |
| **Время** | `затрачено N мин на задачу M` | Вписать время |
| | `итого время` | Сводка времени |
| **Отчёты** | `отк отчет` / `отк от` | Открыть последний отчёт |
| | `отк отчет ИмяФайла` | Открыть по имени |
| | `отк отчет N.N` | Открыть по этапу |
| **Правила** | `правила дорожной карты` | Показать правила |
| | `обновить правила дорожной карты` | Создать новую версию плана |
| | `синхронизация` | Синхронизировать проекты |

## 5. Конвенции

### Лимит активных задач
- Максимум **5 активных задач** одновременно
- При создании 6-й — авто-отложить с причиной

### Учёт времени
- Единица: минуты (целое число)
- При старте: `—`, при завершении: `N`, при промежуточном: `~N`
- Итог: `N мин` или `N ч M мин` при N ≥ 60

### Именование файлов
- Вспомогательные: с суффиксом `_vN` (план, инструкция, changelog)
- Основные: без суффикса (проекты, правила)
- Имя HTML-отчёта: `Отчет_N.N_КраткоеОписание.html`

### План версии — полное описание
`План_СистемаДорожныхКарт_vN.md` — самодостаточный документ. Должен содержать **полную спецификацию**, а не только изменения.

## 6. Формат таблиц

### Уровень 1 (сводная)
```
| № | Задача | P | Сложность | % | Статус | Начало | Конец | Время (мин) |
```

### Уровень 2 (этапы)
```
| № | Этап | Описание | Статус | Дата | Время (мин) | Отчёт | Журнал |
```

### Отложенные
```
| № | Задача | Причина | P | Дата |
```

---

## 7. Формат HTML-отчёта

- Шапка `.meta`: endpoint, задача ДК, проект, дата
- Подвал: привязка к задаче N.N
- CSS-классы: `.issue` (красный), `.ok` (зелёный), `.warn` (оранжевый)
